# od.16. 묶음상품·세트상품·사은품 클레임 정책

## 목적
묶음상품·세트상품·사은품이 포함된 주문의 취소·반품·교환 처리 기준을 정의한다.

---

## 1. 묶음상품 클레임

### 원칙: 구성품 단위 부분 클레임 가능

묶음상품은 구성품 각각이 `od_order_item`으로 등록되므로 개별 클레임 가능.

### 취소 (CANCEL)

| 클레임 범위 | 처리 |
|---|---|
| 구성품 일부 취소 | 해당 `od_order_item` 취소. 안분율 기준 환불 계산 |
| 묶음 전체 취소 | 모든 구성품 `od_order_item` 취소 |

**부분 취소 환불 계산**:
```
묶음가 25,000원, A(40%)=10,000원, B(60%)=15,000원
A만 취소 시 → 환불액 = 10,000원 (bundle_price_rate 기준)
쿠폰 할인 안분: 쿠폰할인 5,000원이었다면 A 귀책 2,000원 차감 → 환불 8,000원
```

### 반품 (RETURN)

- 구성품 개별 반품 가능
- 반품 수거지: 해당 구성품의 `dliv_tmplt.return_addr` 기준
- 반품 배송비: 구성품 귀책에 따라 개별 판단

### 교환 (EXCHANGE)

- 구성품 개별 교환 가능
- 교환 대상: 동일 구성품의 다른 옵션 (다른 상품으로 교환 불가)

### 묶음 할인 조건 파괴 시

- 구성품 1개 취소로 **묶음 최소 구성 수 미달** 시:
  - 나머지 구성품 가격을 정상 `unit_price`로 재계산
  - 묶음 할인 차액 추가 청구 또는 자동 조정

---

## 2. 세트상품 클레임

### 원칙: 세트 전체 단위로만 클레임 가능

```
❌ 세트 내 구성품 하나만 반품 — 불가
✅ 세트 전체 반품 — 가능
```

### 취소

- 세트 `od_order_item` 1개 전체 취소
- 환불액 = 세트 판매가 - 할인 안분 - 적립금 사용 안분

### 반품

- 세트 전체 반품
- 반품 수거지: 세트 `dliv_tmplt.return_addr`
- 세트 전체가 미개봉·완전한 상태여야 반품 가능

### 교환

- 세트 전체 교환
- 동일 세트 상품의 다른 옵션 교환만 가능

---

## 3. 사은품 클레임

### 사은품은 독립 클레임 대상 아님

사은품 자체를 반품/교환하는 것이 아니라, **사은품 지급 조건이 되는 주문**의 클레임에 연동됨.

### 주문 전체 취소 시

```
1. od_order_item(사은품, unit_price=0) 취소
2. pm_gift_issue.gift_issue_status_cd = CANCELLED
3. pm_gift.gift_stock + 1 복원
4. 환불액에 사은품 금액 포함 없음 (0원이므로)
```

### 일부 취소·반품으로 사은품 지급 조건 파괴 시

```
예) 30,000원 이상 주문 시 사은품 지급
    20,000원어치만 취소하여 잔여 주문금액 10,000원 (30,000원 미만)
    → 사은품 지급 조건 파괴
```

처리 방법 (운영 정책으로 선택):

| 방법 | 설명 |
|---|---|
| **회수** | 사은품 반납 요청. `pm_gift_issue.gift_issue_status_cd = CANCELLED` |
| **시가 청구** | 사은품 시가를 `od_claim.add_shipping_fee` 유사 방식으로 추가 차감 |
| **유지** | 정책적으로 사은품 유지 허용 (간소화) |

> 기본 정책: **시가 청구** — 회수 처리 복잡도 대비 고객 경험 개선

### 반품으로 조건 파괴 시

- 동일 처리. `od_claim.claim_cancel_reason_cd` 에 사은품 처리 내역 메모

### 교환 시

- 사은품 유지 (교환은 동일 주문 내 상품 변경이므로 조건 유지)

---

## 4. 공통 — 클레임 시 사은품 연동 체크리스트

클레임 처리 로직에서 아래 순서로 검사:

```
1. 클레임 대상 order_item 확인
2. 해당 order의 gift_issue 조회 (pm_gift_issue)
3. 취소 후 잔여 주문금액 계산
4. 잔여금액 < pm_gift.min_order_amt → 조건 파괴
5. 조건 파괴 시 → 회수 or 시가 청구 처리
6. pm_gift_issue.status 업데이트
7. gift_stock 복원 (회수 확정 시)
```
