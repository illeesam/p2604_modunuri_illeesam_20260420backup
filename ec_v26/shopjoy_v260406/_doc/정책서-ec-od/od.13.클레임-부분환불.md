# od.12. 클레임 - 부분환불 정책

## 목적
부분취소·부분반품·부분교환 완료 시 환불 수단 우선순위, 환불금액 계산, 쿠폰·적립금 처리 정책 정의

## 범위
- 환불 수단 우선순위 (현금성 결제 우선)
- 부분환불 금액 계산 공식
- 쿠폰 처리 (취소 후 재발급 vs 안분 차감)
- 적립금 부분 복원 및 유효기간 정책
- od_pay 부분 환불 레코드 처리

---

## 1. 환불 수단 우선순위

### 1-1. 우선순위 원칙
> **현금성 결제를 먼저 환불** → 카드보다 환불 처리가 빠르고 소비자 보호에 유리

| 우선순위 | 결제수단 | 환불 방식 | 소요기간 |
|---------|---------|---------|---------|
| 1 | 적립금 사용분 | 적립금 원 형태로 복원 (현금 환불 없음) | 즉시 |
| 2 | 캐시(충전금) 사용분 | 캐시 잔액으로 복원 (현금 환불 없음) | 즉시 |
| 3 | 무통장입금 | 고객 등록 계좌로 이체 | 1~3 영업일 |
| 4 | 가상계좌 | 고객 등록 계좌로 이체 | 1~3 영업일 |
| 5 | 간편결제 (카카오페이·네이버페이) | PG사를 통한 자동 환불 | 1~3 영업일 |
| 6 | 핸드폰 결제 | 통신사를 통한 환불 | 3~5 영업일 |
| 7 | 신용카드·체크카드 | 카드사 취소·매출 취소 | 3~7 영업일 |

### 1-2. 복수 결제수단 혼용 시 처리
주문당 N건 결제가 가능한 구조(`od_pay`의 order_id별 N건)이므로:
```
환불_총액 배분:
  Step 1. 적립금 사용분 먼저 복원 (pm_save_usage 취소)
  Step 2. 캐시 사용분 복원 (pm_cache_usage 취소)
  Step 3. 잔여 환불금액을 현금성 결제 → 카드 순으로 차감
  Step 4. 각 od_pay의 refund_amt 업데이트
```

환불금액이 특정 결제 건의 결제금액보다 크면 해당 건 전액 환불 후 잔액을 다음 우선순위 결제 건에서 차감.

## 2. 부분환불 금액 계산

### 2-1. 계산 공식
```
부분환불액 (현금 지급분)
  = 반품상품금액합                          -- Σ(unit_price × claim_qty)
  - 주문쿠폰_안분액                         -- 주문쿠폰 비례 차감
  - 상품쿠폰_할인액                         -- 적용된 쿠폰 할인 (해당 item)
  - 즉시할인_안분액                         -- 상품 즉시할인 비례 차감
  - 배송비_재계산_차액                       -- 무료→유료 전환 시
  - 반품배송비_고객부담분                    -- 반품사유별 (od.05 §3 기준)
  - 상품손상_감액                           -- 검수 감액 (있는 경우)
```

> 적립금·캐시 사용분은 현금으로 환불하지 않고 원 형태 복원 처리 (위 계산에서 제외)

### 2-2. 계산 예시
```
[사례] 3개 중 2개 부분반품, 단순변심
  unit_price       = 20,000원
  order_qty        = 3
  claim_qty        = 2
  
  반품상품금액      = 20,000 × 2 = 40,000원
  주문쿠폰(5,000원 할인, 전체주문금액 60,000원 기준)
    안분액          = 5,000 × (40,000 / 60,000) = 3,333원
  반품배송비 고객부담 = 3,000원 (단순변심 편도)
  
  부분환불액        = 40,000 - 3,333 - 3,000 = 33,667원
```

## 3. 쿠폰 처리 정책

### 3-1. 쿠폰 유형 구분
| 쿠폰 유형 | 정의 | 부분반품 처리 |
|----------|------|-------------|
| 상품쿠폰 | 특정 order_item에 직접 적용 | § 3-2 참조 |
| 주문쿠폰 | 주문 전체 합계에서 차감 | § 3-3 참조 |

### 3-2. 상품쿠폰 처리
| 상황 | 처리 |
|------|------|
| 쿠폰 적용 item **전량** 반품 | 쿠폰 **재발급** → § 3-4 유효기간 정책 |
| 쿠폰 적용 item **부분수량** 반품 | 쿠폰 재발급 없음, 환불금액에서 `쿠폰할인액 × (claim_qty/order_qty)` 차감 |
| 쿠폰 미적용 item 반품 | 쿠폰 영향 없음 |

### 3-3. 주문쿠폰 처리
```
주문쿠폰_안분액 = 주문쿠폰_총할인액 × (반품item_금액합 / 전체주문_상품금액합)
```
- 안분액은 환불금액에서 차감
- **쿠폰 자체는 재발급하지 않음** — 주문이 일부 유효하므로
- **예외: 전체 반품·취소 완료 시** → 주문쿠폰 재발급

### 3-4. 쿠폰 재발급 유효기간
| 조건 | 재발급 유효기간 |
|------|--------------|
| 원 쿠폰이 아직 유효하고 잔여 > 7일 | 원 만료일 그대로 유지 |
| 원 쿠폰 잔여 ≤ 7일 | 재발급일로부터 30일 |
| 원 쿠폰 이미 만료 | 관리자 재량으로 30일 신규 쿠폰 발급 (원 쿠폰 조건 동일) |

### 3-5. 쿠폰 재발급 처리 방법
- **취소 방식이 아닌 재발급**: 원 `pm_coupon_issue` 레코드는 USED 상태 유지, 신규 `pm_coupon_issue` 레코드를 ACTIVE로 생성
- 재발급 사유: `issue_reason = 'CLAIM_RETURN'` (또는 CANCEL)
- `od_claim_id` 참조 저장

## 4. 적립금(포인트) 부분 복원 정책

### 4-1. 복원 대상 적립금
1. **구매 시 사용한 적립금** (`pm_save_usage`)
2. **구매 완료 후 지급된 적립금** (`pm_save_issue` — 이미 CONFIRMED된 경우)

### 4-2. 사용 적립금 복원 (부분환불)
```
복원_적립금액 = 주문시_총사용적립금 × (반품item_금액합 / 전체주문_상품금액합)
```
- `pm_save_usage` 레코드 중 해당 order_id 건: 사용금액에서 복원_적립금액 차감
- `pm_save` 원장에 `save_type_cd = CANCEL` 타입으로 복원 기록
- `mb_member.accumulation_balance` 잔액 업데이트

### 4-3. 적립금 복원 유효기간
| 조건 | 복원 후 유효기간 |
|------|---------------|
| 원 적립금 만료일까지 **8일 이상** | 원 만료일 그대로 유지 |
| 원 적립금 만료일까지 **7일 이하** | 복원일로부터 30일 연장 |
| 원 적립금 이미 **소멸 처리** 완료 | 복원일로부터 30일 유효한 신규 적립금 생성 (관리자 재량) |

### 4-4. 지급 예정 적립금 취소
- 반품 완료 시점에 `pm_save_issue.save_issue_status_cd = CANCELLED` 처리
- 부분반품: 해당 order_item의 적립 예정 금액만 취소
- 전량반품: 해당 주문 전체 적립 예정 취소

### 4-5. 적립 기준 재계산
- 부분반품 완료 후 잔여 주문의 실결제금액으로 재계산하지 않음
- 이미 지급·확정된 적립금은 유지, 반품분에 해당하는 적립금만 취소

---

## 5. od_pay 부분환불 처리

### 5-1. 레코드 업데이트 방식
- 기존 `od_pay` 레코드의 `refund_amt` 누적 업데이트 (신규 행 추가가 아닌 컬럼 업데이트)
- `refund_status_cd`: PENDING → COMPLT
- `refund_date`: 환불 처리 완료 시각
- `pay_status_cd`: 
  - 부분환불 완료: `PARTIAL_REFUND`
  - 전액 환불 완료: `REFUNDED`

### 5-2. 부분환불 가능 조건
- `pay_amt - 기존_refund_amt > 0` (환불 가능 잔액 존재)
- `pay_status_cd IN ('COMPLT', 'PARTIAL_REFUND')` (결제 완료 상태)

---

## 6. 환불 처리 타임라인

| 단계 | 처리 내용 | 시점 |
|------|---------|------|
| 클레임 접수 | 환불 예정금액 산출, 클레임 생성 | 신청 즉시 |
| 클레임 승인 | 환불 금액 확정 | 관리자 승인 |
| 입고 완료 | 실제 환불 실행 (현금성) | 검수 후 |
| 적립금·캐시 복원 | 즉시 복원 (클레임 COMPLT 시) | 클레임 완료 |
| 카드 취소 | PG사 취소 요청 → 영업일 기준 | 클레임 완료 |

---

## 관련 테이블
- `od_pay(결제)` — 결제 마스터 (refund_amt 누적, pay_status_cd: PARTIAL_REFUND/REFUNDED 전환)
- `od_refund(환불)` — 환불 건별 마스터 (total_refund_amt, refund_prod_amt, refund_coupon_amt, refund_ship_amt)
- `od_refund_method(환불수단 내역)` — 수단별 환불금액 행 (refund_priority: 1=카드, 2=캐쉬, 3=적립금)
- `od_order_discnt(주문할인·차감 내역)` — 주문쿠폰 안분 기준 / 적립금·캐쉬 복원 상태 추적 (restore_yn, restore_amt)
- `od_order_item_discnt(주문상품할인 내역)` — 상품쿠폰 안분 계산 원본
- `pm_coupon_issue` — 쿠폰 재발급 처리
- `pm_coupon_usage` — 쿠폰 사용 이력
- `pm_save` — 적립금 원장
- `pm_save_usage` — 적립금 사용 이력
- `pm_save_issue` — 적립금 지급 이력
