# od.13. 클레임 - 부분교환 정책

## 목적
하나의 주문에서 일부 상품(order_item) 또는 일부 수량만 교환하는 정책 정의

## 범위
- order_item 단위 부분교환 신청
- 수량(claim_qty) 기반 부분교환
- 교환 배송비 재계산 (왕복)
- 교환상품 가격차 처리 (추가결제·환불)
- 쿠폰·할인·적립금 안분 처리

---

## 1. 부분교환 신청 기준

### 1-1. 신청 가능 단위
- **상품 단위**: 주문 내 특정 order_item만 교환 신청
- **수량 단위**: 동일 order_item 내에서 일부 수량만 교환 가능
  - 예: order_qty=3 중 claim_qty=2 교환

### 1-2. 신청 가능 수량 계산
```
신청가능_최대수량 = order_qty - cancel_qty - 기존_진행중_claim_qty
```
- `cancel_qty`: 이미 취소·반품 완료된 수량
- 최소 신청수량: 1개

### 1-3. 교환 신청 불가 조건
| 조건 | 사유 |
|------|------|
| order_item_status_cd = CANCEL | 이미 전량 취소 |
| claim_item_status_cd = REQUESTED/APPROVED/PROCESSING | 진행중인 클레임 있음 |
| 신청가능_최대수량 ≤ 0 | 남은 수량 없음 |
| 위생상품 / 오염·손상 확인 | 교환 불가 상품 |
| 교환기한(배송완료 후 30일) 초과 | 기한 만료 |

---

## 2. 부분교환 처리 프로세스

```
신청 → 승인 → 수거중(IN_PICKUP) → 수거완료 → 입고(검수)
    → [추가결제 필요 시] 추가결제 요청/완료
    → 교환상품 배송(IN_TRANSIT) → 교환완료(COMPLT)
```

### 2-1. 신청 시 생성 데이터
- `od_claim` 1건 생성 (`claim_type_cd = EXCHANGE`)
- `od_claim_item` N건 생성
  - `claim_qty`: 교환 신청 수량
  - `item_amt`: `unit_price × claim_qty`
- 교환 희망 상품 정보 기록 (상품ID, 옵션, 수량)

### 2-2. 교환완료 시 od_order_item 처리
교환 완료 후 원 order_item 상태 업데이트:
```sql
UPDATE od_order_item SET
  order_item_status_cd = CASE
    WHEN claim_qty >= 신청가능_최대수량 THEN 'EXCHANGE_COMPLT'  -- 전량 교환
    ELSE 'PARTIAL_EXCHANGE_COMPLT'                              -- 부분 교환
  END
WHERE order_item_id = ?;
```
교환상품은 새로운 `od_order_item` 레코드 생성 (기존 order_id에 연결, order_item_status_cd = EXCHANGE_NEW)

## 3. 교환 배송비 정책

### 3-1. 배송비 부담 구분 (사유별)
| 사유 | 수거배송비 | 발송배송비 |
|------|----------|----------|
| 상품하자(DEFECT) | 판매자 100% | 판매자 100% |
| 배송손상(DAMAGE) | 판매자 100% | 판매자 100% |
| 오배송(WRONG_ITEM) | 판매자 100% | 판매자 100% |
| 사이즈/색상오류 | 판매자 100% | 판매자 100% |
| 단순변심(CHANGE_OF_MIND) | 고객 50% / 판매자 50% | 고객 부담 |
| 기타(OTHER) | 협의 | 협의 |

### 3-2. 부분교환 배송비 계산
```
교환배송비 = 수거배송비 + 발송배송비
고객부담_배송비 = 교환배송비 × 고객부담비율
```
- 고객부담 배송비는 추가결제 요청 또는 환불금액에서 차감

### 3-3. 추가결제 필요 여부 판단
| 상황 | 처리 |
|------|------|
| 교환상품 가격 = 원 상품 가격, 배송비 판매자 부담 | 추가결제 없음 |
| 교환상품 가격 > 원 상품 가격 | 가격차 만큼 추가결제 요청 |
| 고객부담 배송비 발생 | 배송비 추가결제 요청 |
| 교환상품 가격 < 원 상품 가격 | 가격차 환불 또는 캐시 지급 |

---

## 4. 교환상품 가격차 처리

### 4-1. 교환상품이 더 비쌀 경우
- 차액 = `교환상품_unit_price × claim_qty - 원상품_unit_price × claim_qty`
- 추가결제 요청 생성 (→ [od.14.클레임-추가결제.md])
- 추가결제 완료 후 교환상품 발송 처리

### 4-2. 교환상품이 더 저렴할 경우
- 차액 환불 또는 캐시 적립 중 고객 선택
- 기본 정책: 차액을 캐시(`pm_cache`)로 지급 (즉시 처리 가능)
- 고객 요청 시 원 결제수단으로 환불 (환불 우선순위 → [od.12] §1 준용)

### 4-3. 가격 동일 + 재고 없음
- 고객 동의 하에 유사상품 교환 또는 반품 전환
- 반품 전환 시: od_claim.claim_type_cd = RETURN으로 변경 (관리자 처리)

---

## 5. 쿠폰·할인 안분 처리

### 5-1. 교환은 환불 아님 — 쿠폰 재발급 원칙
- 교환은 동일 가치 상품으로 대체이므로 기본적으로 쿠폰 재발급 없음
- 단, 교환상품 가격차 발생 시 정산 필요

### 5-2. 상품쿠폰 처리
| 상황 | 처리 |
|------|------|
| 교환 item에 상품쿠폰 적용됨, 동일 가격 교환 | 쿠폰 할인 유지 (교환상품에 쿠폰 이월) |
| 교환 item에 상품쿠폰 적용됨, 가격 상승 교환 | 원 쿠폰 할인 유지 + 가격차 추가결제 |
| 교환 item에 상품쿠폰 적용됨, 가격 하락 교환 | 쿠폰 비율로 안분하여 환불금액 계산 |

### 5-3. 주문쿠폰 처리
- 부분교환 시 주문 자체는 유효하므로 **주문쿠폰 재발급 없음**
- 가격차 정산 시 쿠폰 안분액을 반영
  - 가격 상승: 추가결제 대상에서 쿠폰 안분액 반영 불요 (쿠폰은 원 주문에 귀속)
  - 가격 하락: 환불금액에서 주문쿠폰 안분액 차감

---

## 6. 적립금 처리

### 6-1. 교환 시 적립금 처리 원칙
- 가격 동일 교환: 적립금 사용분 변동 없음
- 가격 상승 교환: 추가금액에 대한 적립금 재사용 가능 (새 od_pay 연결)
- 가격 하락 교환: 차액에 해당하는 사용 적립금 복원

### 6-2. 구매 완료 후 지급 예정 적립금
- 교환 완료 후 원 order_item의 `pm_save_issue` 취소
- 교환상품(신규 order_item) 기준으로 새로운 `pm_save_issue` 생성
  - 교환상품 금액 기준으로 재계산

---

## 7. 부분교환 제약사항
- 위생상품(속옷·수영복·귀걸이 등) 교환 불가
- 교환 후 동일 item에 추가 교환 불가 (반품은 허용)
- 1회 교환 최대 수량: 5개 (order_item 합산)
- 추가결제 미완료 시 교환상품 발송 보류
- 재고 부족 시 고객 동의 없이 임의 대체 불가

## 8. 상태 코드

### od_order_item.order_item_status_cd
| 상태 | 코드 | 설명 |
|------|------|------|
| 교환완료 | EXCHANGE_COMPLT | 전량 교환 완료 |
| 부분교환완료 | PARTIAL_EXCHANGE_COMPLT | 일부 수량 교환 완료 |
| 교환신상품 | EXCHANGE_NEW | 교환으로 생성된 신규 order_item |

## 관련 테이블
- `od_claim` — 교환 클레임 마스터
- `od_claim_item` — 교환 항목 (claim_qty)
- `od_order_item` — 원 item 상태 업데이트 + 교환상품 신규 생성
- `od_dliv` — 수거 배송 + 교환상품 발송 배송 (2건)
- `pm_cache` — 가격 하락 시 캐시 지급
