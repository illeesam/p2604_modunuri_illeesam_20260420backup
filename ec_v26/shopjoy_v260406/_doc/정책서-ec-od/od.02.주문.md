# 313. 주문 관리 정책

## 목적
주문 생성, 처리, 변경에 대한 정책 정의

## 범위
- 주문 상태 관리 (주문·클레임·결제·배송 전체)
- 주문 결제 관리
- 부분취소·부분반품·부분교환·부분환불
- 주문 배송 추적

---

## 설계 원칙: order_item이 기준 단위

> **부분반품·부분교환·부분배송** 등 item별 독립 처리가 필요하므로
> `od_order_item.order_item_status_cd` 가 실제 라이프사이클의 **소스 오브 트루스**.
> `od_order.order_status_cd` 는 order_item 상태를 집계한 **요약 상태** (빠른 조회·필터용).
>
> 클레임 상태는 별도 테이블(`od_claim_item`)이 독립 관리.
> → order_item 상태와 클레임 상태는 동시에 공존 가능.

---

## 전체 흐름도

### ◆ 정상 주문 흐름

```
od_order
order_status_cd:   PENDING ──► PAID ──► PREPARING ──► SHIPPED ──────────────► COMPLT
                      │           │                                  ▲               ▲
                      │           │                                  │               │
od_order_item                                                   (전체배송완료)  (교환완료 시)
order_item_status_cd:
                   ORDERED ──► PAID ──► PREPARING ──► SHIPPING ──► DELIVERED ──► CONFIRMED
                      │           │                                    │               │
                   (취소가능)  (취소가능)                         (반품·교환 신청가능, 30일)
```

### ◆ 부분클레임 — 두 상태가 공존하는 구조

```
예시: order_qty=3 중 1개 반품 신청

  ┌─ od_order_item ──────────────────────────────────────┐
  │  order_item_status_cd = DELIVERED  ← 상태 그대로 유지 │
  │  order_qty            = 3                             │
  │  cancel_qty           = 0  (반품 완료 후 → 1 누적)    │
  │  item_cancel_amt      = 0  (반품 완료 후 → +20,000)   │
  └──────────────────────────────────────────────────────┘
                  │  order_item_id (FK)
                  ▼
  ┌─ od_claim_item ──────────────────────────────────────┐
  │  claim_item_status_cd = RETURNING  ← 독립적으로 진행  │
  │  claim_qty            = 1                             │
  │  claim_type_cd (via od_claim) = RETURN                │
  └──────────────────────────────────────────────────────┘
```

### ◆ 부분환불 — od_pay 상태 전이

```
  ┌─ od_pay ─────────────────────────────────────────────┐
  │  pay_amt       = 60,000원  (원 결제금액)               │
  │  refund_amt    = 0  → 20,000원 (부분반품 후 누적)       │
  │                   → 60,000원 (전량 반품 후 전액환불)    │
  │                                                       │
  │  pay_status_cd:  COMPLT ──► PARTIAL_REFUND ──► REFUNDED │
  └──────────────────────────────────────────────────────┘
```

### ◆ 취소 흐름

```
  ORDERED ──► CANCEL_REQ ──► CANCELLED   (결제 전)
  PAID    ──► CANCEL_REQ ──► CANCELLED   (배송 전)
  PREPARING  취소 불가  (고객센터 문의)
  SHIPPING   취소 불가  (반품으로 처리)
```

### ◆ 반품 흐름

```
  DELIVERED ──► (od_claim 생성, RETURN)
  CONFIRMED ──►
                  REQUESTED ──► APPROVED ──► IN_PICKUP ──► PROCESSING ──► COMPLT
                                                                  │
                                                          (입고검수 완료 시)
                                                          cancel_qty += claim_qty
                                                          od_pay.refund_amt 누적
                                                          (전량이면 order_item → CANCELLED)
```

### ◆ 교환 흐름

```
  DELIVERED ──► (od_claim 생성, EXCHANGE)
  CONFIRMED ──►
                  REQUESTED ──► APPROVED ──► IN_PICKUP ──► IN_TRANSIT ──► COMPLT
                                                                │
                                                    (교환상품 배송 완료 시)
                                                    신규 od_order_item 생성 (DELIVERED)
                                                    (가격차 있으면 od_pay 추가결제)
```

## 상태 코드 정의

### `od_order.order_status_cd` — 주문 집계 상태 (od_order_item 요약)

| 주문 | 취소 | 반품 | 교환 | 코드값 | 코드라벨 | 비고 |
|:---:|:---:|:---:|:---:|--------|---------|------|
| O | - | - | - | PENDING   | 결제대기   | 주문 생성. 24시간 내 미결제 시 자동 취소 |
| O | - | - | - | PAID      | 결제완료   | 결제 승인 완료 |
| O | - | - | - | PREPARING | 배송준비중 | 1개 이상 item PREPARING 상태 |
| O | - | - | O | SHIPPED   | 배송중     | 1개 이상 item SHIPPING 또는 교환상품 발송 중 |
| O | - | - | O | COMPLT    | 완료       | 전체 item DELIVERED 이상 / 교환 완료 후 최종 상태 |
| - | O | - | - | CANCELLED | 취소완료   | 전체 item 취소 완료 (cancel_qty = order_qty) |
| - | - | O | - | RETURNED  | 반품완료   | 전체 item 반품 완료 |

> 부분취소·부분반품 진행 중 order_status_cd는 잔여 활성 item의 가장 앞선 상태를 반영.
> 예) 3개 중 1개 반품 중 → order_status_cd = SHIPPED (나머지 2개 기준)

---

### `od_order_item.order_item_status_cd` — 품목 주문 상태 (소스 오브 트루스, 주문 흐름 전용)

> 클레임 진행 중이어도 이 컬럼은 주문 흐름 상태를 유지.
> 전량 취소·반품 완료(cancel_qty = order_qty) 시에만 CANCELLED 로 전환.
> 교환 완료 후 교환상품은 신규 order_item(DELIVERED)으로 생성 — 원 item은 변동 없음.

| 주문 | 취소 | 반품 | 교환 | 코드값 | 코드라벨 | 비고 |
|:---:|:---:|:---:|:---:|--------|---------|------|
| O | - | - | - | ORDERED   | 주문완료   | 초기값. 결제 대기 중 |
| O | - | - | - | PAID      | 결제완료   | 결제 승인 완료 |
| O | - | - | - | PREPARING | 배송준비중 | 판매자 출고 준비 시작 |
| O | - | - | O | SHIPPING  | 배송중     | 송장번호 등록 / 교환상품 신규 item도 이 상태 통과 |
| O | - | - | O | DELIVERED | 배송완료   | 택배사 배송완료 스캔 / 교환상품 수령 완료 |
| O | - | - | O | CONFIRMED | 구매확정   | 고객 수동 확정 또는 배송완료 7일 후 자동 확정 |
| - | O | O | - | CANCELLED | 전량취소   | cancel_qty = order_qty 시. 취소·반품 완료 모두 해당 |

---

### `od_claim.claim_status_cd` — 클레임 마스터 상태

| 취소 | 반품 | 교환 | 코드값 | 코드라벨 | 비고 |
|:---:|:---:|:---:|--------|---------|------|
| O | O | O | REQUESTED  | 요청       | 클레임 신청 접수. 관리자 검토 대기 |
| O | O | O | APPROVED   | 승인       | 관리자 승인. 수거 또는 환불 절차 시작 |
| - | O | O | IN_PICKUP  | 수거중     | 반품·교환 상품 수거 진행 중 (취소는 물리적 수거 없음) |
| - | O | - | PROCESSING | 검수중     | 반품 입고 후 상품 검수 진행 중 |
| - | - | O | IN_TRANSIT | 교환발송중  | 교환상품 배송 중 |
| O | O | O | COMPLT     | 완료       | 클레임 처리 완료. 환불·교환 확정 |
| O | O | O | REJECTED   | 거절       | 기한초과·불가상품 등으로 클레임 거절 |
| O | O | O | CANCELLED  | 철회       | 고객이 직접 클레임 취소 |

### `od_claim.claim_type_cd` — 클레임 유형

| 코드값 | 코드라벨 | 비고 |
|--------|---------|------|
| CANCEL   | 취소 | 배송 시작 전 주문 취소 |
| RETURN   | 반품 | 배송 완료 후 반품 |
| EXCHANGE | 교환 | 배송 완료 후 교환 |

### `od_claim_item.claim_item_status_cd` — 클레임 항목 상태 (order_item과 독립 공존)

> order_item 1건에 대해 부분수량 클레임 가능 (claim_qty ≤ order_qty - cancel_qty).
> claim_item 진행 중이어도 order_item.order_item_status_cd 는 유지됨.

| 취소 | 반품 | 교환 | 코드값 | 코드라벨 | 비고 |
|:---:|:---:|:---:|--------|---------|------|
| O | O | O | REQUESTED  | 요청       | 클레임 항목 신청 접수. 초기값 |
| O | O | O | APPROVED   | 승인       | 관리자 승인 |
| - | O | O | IN_PICKUP  | 수거중     | 수거 진행 중 (취소는 해당 없음) |
| - | O | - | PROCESSING | 검수중     | 반품 입고 후 검수 (반품 전용) |
| - | - | O | IN_TRANSIT | 교환발송중  | 교환상품 배송 중 (교환 전용) |
| O | O | O | COMPLT     | 완료       | 클레임 항목 처리 완료. cancel_qty/item_cancel_amt 누적 |
| O | O | O | REJECTED   | 거절       | 해당 항목 클레임 불가 |
| O | O | O | CANCELLED  | 철회       | 고객이 해당 항목 클레임 취소 |

#### 수량·금액 추적 필드 (od_order_item)

| 필드 | 설명 | 부분클레임 시 변화 |
|------|------|-----------------|
| `order_qty` | 원 주문 수량 | 변동 없음 |
| `cancel_qty` | 취소·반품 완료 누적 수량 | COMPLT 시 += claim_qty |
| `item_order_amt` | 원 주문 금액 (unit_price × order_qty) | 변동 없음 |
| `item_cancel_amt` | 취소·반품 완료 누적 금액 | COMPLT 시 += 환불확정금액 |
| `item_completed_amt` | 완료 금액 (item_order_amt - item_cancel_amt) | COMPLT 시 재계산 |

### `od_pay.pay_status_cd` — 결제 상태

| 주문 | 취소 | 반품 | 교환 | 코드값 | 코드라벨 | 비고 |
|:---:|:---:|:---:|:---:|--------|---------|------|
| O | - | - | - | PENDING        | 결제대기  | 결제 요청 후 승인 대기 |
| O | - | - | O | COMPLT         | 결제완료  | 결제 승인 완료 / 교환 추가결제 완료 |
| O | - | - | - | FAILED         | 결제실패  | PG 승인 실패 |
| - | O | - | - | CANCELLED      | 결제취소  | 취소에 의한 전액 결제 취소 |
| - | O | O | O | PARTIAL_REFUND | 부분환불  | 부분취소·부분반품·교환 가격차 환불 완료 |
| - | O | O | - | REFUNDED       | 전액환불  | 전액 환불 완료 (전체 취소·전체 반품) |

> **부분환불 흐름**: COMPLT → PARTIAL_REFUND (부분) → REFUNDED (전액 완료 시)
> 교환 가격차 추가결제는 신규 od_pay 레코드 생성 (기존 건은 유지)

### `od_dliv.dliv_status_cd` — 배송 상태

> 출고(outbound) 배송과 반품·교환 수거(inbound) 배송 모두 이 코드 사용.
> od_claim.return_status_cd / od_claim.inbound_dliv_id 로 구분.

| 주문 | 취소 | 반품 | 교환 | 코드값 | 코드라벨 | 비고 |
|:---:|:---:|:---:|:---:|--------|---------|------|
| O | - | - | O | READY      | 출고준비   | 출고 준비 중 / 교환상품 출고 준비 |
| O | - | O | O | PICKED     | 픽업완료   | 택배사 픽업 완료 / 반품·교환 수거 픽업 |
| O | - | O | O | IN_TRANSIT | 배송중     | 택배사 배송 진행 / 반품 수거 이동 / 교환상품 발송 |
| O | - | - | O | DELIVERED  | 배송완료   | 수령자 인수 완료 / 교환상품 수령 완료 |
| - | - | O | O | RETURN_REQ | 반품요청   | 반품·교환 수거 요청 접수 |
| - | - | O | O | RETURNED   | 수거완료   | 반품·교환 상품 입고 완료 |

## 주요 정책

### 1. 주문 생성
- **필수항목**: 상품(최소 1개), 수량, 수령자 정보(이름·연락처·주소)
- **초기상태**: `od_order.order_status_cd = PENDING`, `od_order_item.order_item_status_cd = ORDERED`
- **자동만료**: 미결제 상태 24시간 후 자동 취소

### 2. 주문 가격 정책

#### 2-1. 금액 계산
- **상품합계금액**: 상품 판매가 × 수량 (옵션 추가금액 포함)
- **할인금액**: 쿠폰할인, 카테고리할인 적용
- **배송금액**: 판매사별 독립 계산
  | 판매사수 | 기본배송료 | 합계 |
  |---------|----------|-----|
  | 1개 | 3,000원 | 3,000원 |
  | 2개 | 3,000원 × 2 | 6,000원 |
  | 3개 | 3,000원 × 3 | 9,000원 |
- **실결제금액**: 상품합계금액 - 할인금액 + 배송금액

#### 2-2. 적립금
- **지급**: 배송완료 후 5일 (반품 미발생 확인 후) 회원등급별 1~2.5% (BASIC 1% / SILVER 1.5% / GOLD 2% / VIP 2.5%, 쿠폰·할인금·배송료 제외, 상세는 [pm.04] 참조)

### 3. 부분클레임 처리 원칙
- **신청 단위**: order_item 단위, 수량(claim_qty) 단위로 부분 신청 가능
- **상태 분리**: order_item_status_cd(주문 흐름)와 claim_item_status_cd(클레임 흐름)는 독립 유지
- **수량 검증**: `claim_qty ≤ order_qty - cancel_qty - 진행중_claim_qty`
- **CANCELLED 전환**: `cancel_qty = order_qty` 달성 시에만 order_item → CANCELLED

### 4. 부분배송
- **지원여부**: 여러 판매자 상품은 부분배송 가능
- **배송료**: 각 배송건별 독립 계산
- **추적**: od_order_item별 dliv_tracking_no 저장

### 5. 주문 취소 정책
- **결제전(ORDERED)**: 고객 직접 취소 가능, 즉시 처리
- **결제후(PAID)**: 배송 전까지만 가능, 환불 처리
- **PREPARING 이후**: 취소 불가 → 반품으로 처리
- **부분취소**: order_item 단위로 취소 신청, 배송비 재계산

## 환불 수단 우선순위

## 데이터 시뮬레이션: 복합할인 주문 → 2회 부분반품

> **시나리오명**: 주문 → 1차반품 → 2차반품
> 복합 할인(상품할인 + 주문쿠폰 + 배송비할인) + 복합 결제(적립금+캐쉬+카드) 환경에서
> 부분반품이 2회 발생할 때 **환불 우선순위(카드→캐쉬→적립금)** 적용 금액 흐름 검증.

### Step 1 — 주문

#### 주문 상품 구성
| 항목 | 상품1 | 상품2 |
|---|---|---|
| 개당 정가 | 10,000원 | 10,000원 |
| 주문수량 | 5개 | 5개 |
| 정상가 소계 | 50,000원 | 50,000원 |
| **상품 즉시할인** | △5,000원 (금액할인) | △5,000원 (10% 할인) |
| 할인 후 소계 | 45,000원 | 45,000원 |

#### 주문 금액 계산
| 항목 | 금액 | 비고 |
|---|---|---|
| 상품 소계 | 90,000원 | 45,000 + 45,000 |
| 주문할인쿠폰 | △9,000원 | 5만원↑ 10% → 90,000 × 10% |
| 상품 결제금액 | 81,000원 | |
| 배송비 | 3,000원 | |
| 배송비 할인 | △3,000원 | 배송비 할인 쿠폰 |
| **최종 결제금액** | **81,000원** | |

#### 결제수단
| 순위 | 수단 | 금액 |
|---|---|---|
| 1순위 (환불 선순위) | 카드 | 41,000원 |
| 2순위 | 캐쉬 | 20,000원 |
| 3순위 | 적립금 | 20,000원 |
| | **합계** | **81,000원** |

#### 주문쿠폰 안분 후 상품별 확정 단가
> 주문쿠폰(9,000원)을 상품 금액 비율(50% : 50%)로 안분

| | 상품1 | 상품2 |
|---|---|---|
| 할인 후 소계 | 45,000원 | 45,000원 |
