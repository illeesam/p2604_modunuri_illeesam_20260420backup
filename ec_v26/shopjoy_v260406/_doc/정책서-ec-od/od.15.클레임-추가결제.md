# od.14. 클레임 - 추가결제 정책

## 목적
클레임(반품·교환) 처리 과정에서 발생하는 추가 비용(배송비·가격차·구매금액)에 대한 추가결제 요청 및 처리 프로세스 정의

## 범위
- 추가결제 발생 사유
- 추가결제 요청 프로세스 (관리자 일괄작업 → 결재처리)
- 추가결제 방법 (신규 od_pay 생성)
- od_claim.appr_* 필드 활용
- 추가결제 미완료 시 클레임 처리 보류 정책

---

## 1. 추가결제 발생 사유

### 1-1. 배송비 추가결제
| 발생 상황 | 추가금액 대상 |
|----------|-------------|
| 부분반품으로 무료배송 조건 미충족 | 배송비 재계산 차액 |
| 반품 배송비 고객 부담 (단순변심 등) | 왕복 배송비 × 고객부담비율 |
| 교환 배송비 고객 부담 | 왕복 배송비 × 고객부담비율 |
| 도서산간 추가 배송비 | 실비 + 추가요금 |

### 1-2. 교환상품 가격차 추가결제
| 발생 상황 | 추가금액 대상 |
|----------|-------------|
| 교환 요청 상품이 원 상품보다 비쌀 때 | (교환상품 단가 - 원 상품 단가) × claim_qty |
| 교환 후 사이즈·색상 변경으로 가격 상승 | 가격 상승분 |

### 1-3. 추가 구매 금액
| 발생 상황 | 추가금액 대상 |
|----------|-------------|
| 교환 불가 → 유사상품 대체 (가격 상승) | 가격차 |
| 관리자 판단에 의한 추가비용 발생 | 사유 명시 후 요청 |

---

## 2. 추가결제 요청 프로세스

### 2-1. 관리자 처리 흐름
```
[관리자 클레임 목록]
  → 해당 클레임 행 체크 선택
  → 상단 "📝 변경작업 선택" 버튼 클릭
  → "추가결재요청" 탭 선택
  → 요청 정보 입력
  → 요청 생성 (od_claim.appr_* 업데이트)

[시스템]
  → 회원에게 알림/이메일 발송 (추가결제 안내)

[회원]
  → 마이페이지 or 알림 링크에서 추가결제 진행
  → 추가결제 완료 시 클레임 처리 재개

[관리자]
  → 추가결제 확인 후 클레임 다음 단계 진행
```

### 2-2. 추가결제 요청 입력 항목
| 항목 | 필드 | 설명 |
|------|------|------|
| 요청 담당자 | `appr_req_user_id` | sy_user.user_id |
| 요청일시 | `appr_req_date` | TIMESTAMP |
| 결재상태 | `appr_status_cd` | REQ (초기값) |
| 요청금액 | `appr_amt` | BIGINT |
| 요청대상 구분 | `appr_target_cd` | ORDER / PROD / DLIV / EXTRA |
| 요청대상명 | `appr_target_nm` | 수정 가능 (예: "단순변심 반품 배송비") |
| 사유/메모 | `appr_reason` | 최대 500자 |

### 2-3. 알림 템플릿 치환 변수
```
{target}    → appr_target_cd 한글명 (배송비 / 교환금액차 / 추가구매금액)
{targetNm}  → appr_target_nm (예: "교환상품 가격차 (사이즈 L→XL)")
{amount}    → appr_amt (원 단위 포맷)
{reason}    → appr_reason
```

---

## 3. 결재 상태 관리 (appr_status_cd)

| 상태 | 코드 | 설명 | 다음 상태 |
|------|------|------|---------|
| 요청 | REQ | 추가결제 요청 생성 | APPROVED / REJECTED |
| 승인 | APPROVED | 고객 결제 완료 확인 | DONE |
| 거절 | REJECTED | 고객 거절 또는 취소 | (클레임 철회 또는 재협의) |
| 완료 | DONE | 추가결제 처리 완료, 클레임 다음 단계 진행 | — |

---

## 4. 추가결제 방법 (신규 od_pay 생성)

### 4-1. 추가결제 처리 방식
- 원 `od_pay` 레코드와 **별도**로 신규 `od_pay` 레코드 생성
- `od_pay.order_id`: 기존 주문 ID 동일 사용
- `od_pay.memo`: `"클레임추가결제:claim_id"` 형식으로 연결 정보 기록

### 4-2. 추가결제 수단
| 결제수단 | 허용 여부 | 비고 |
|---------|---------|------|
| 카드 결제 | ✅ | PG사 신규 결제 |
| 카카오페이 | ✅ | 간편결제 |
| 네이버페이 | ✅ | 간편결제 |
| 무통장입금 | ✅ | 3일 내 입금 확인 필요 |
| 가상계좌 | ✅ | 별도 가상계좌 발급 |
| 캐시(충전금) | ✅ | pm_cache 잔액 사용 |
| 적립금 | ✅ | pm_save 잔액 사용 |
| 핸드폰결제 | ⚠️ | 한도 내에서만 허용 |

### 4-3. 추가결제 완료 시 처리
```sql
-- 신규 od_pay 생성 (추가결제)
INSERT INTO od_pay (pay_id, order_id, pay_method_cd, pay_amt, pay_status_cd, memo, ...)
VALUES (?, ?, ?, appr_amt, 'PENDING', 'CLAIM_EXTRA_PAY:claim_id', ...);

-- 추가결제 완료 시
UPDATE od_claim SET
  appr_status_cd = 'DONE',
  appr_aprv_user_id = ?,
  appr_aprv_date = NOW()
WHERE claim_id = ?;
```

## 5. 추가결제 미완료 시 클레임 처리 보류

### 5-1. 보류 조건
| 클레임 유형 | 보류 시점 | 보류 내용 |
|-----------|---------|---------|
| 반품 | 입고 완료 후 배송비 추가결제 요청 시 | 환불 처리 보류 |
| 교환 | 입고 완료 후 교환상품 발송 전 | 교환상품 배송 보류 |

### 5-2. 보류 해제 조건
- `appr_status_cd = DONE` (추가결제 완료 확인) → 클레임 다음 단계 진행
- `appr_status_cd = REJECTED` (고객 거절)
  - 반품: 배송비 미지급 → 환불금액에서 차감 후 환불 진행
  - 교환: 반품으로 전환하거나 클레임 철회 처리

### 5-3. 보류 자동 해제 기한
- 추가결제 요청일로부터 **7일** 내 미결제 시 관리자에게 알림
- **14일** 내 미결제 시 자동 처리 방침 적용:
  - 반품: 배송비 환불금액 차감 후 강제 환불 처리 (`appr_status_cd = DONE` 처리)
  - 교환: 반품 전환 처리

## 6. 추가결제 관련 알림

### 6-1. 회원 알림
| 시점 | 알림 내용 |
|------|---------|
| 추가결제 요청 생성 | 추가결제 안내 (금액, 사유, 기한) |
| 추가결제 7일 미납 | 독촉 알림 |
| 추가결제 완료 | 완료 확인 + 클레임 다음단계 안내 |

### 6-2. 관리자 알림
| 시점 | 알림 내용 |
|------|---------|
| 추가결제 완료 | 클레임 처리 재개 안내 |
| 7일/14일 미납 | 강제 처리 여부 결정 필요 알림 |

## 7. 주요 필드 (od_claim)

| 필드 | 코드/형식 | 설명 |
|------|---------|------|
| `appr_status_cd` | REQ / APPROVED / REJECTED / DONE | 결재(추가결제) 상태 |
| `appr_status_cd_before` | — | 변경 전 결재상태 |
| `appr_amt` | BIGINT | 추가결제 요청 금액 |
| `appr_target_cd` | ORDER / PROD / DLIV / EXTRA | 결제 대상 구분 |
| `appr_target_nm` | VARCHAR(200) | 결제 대상명 (수정 가능) |
| `appr_reason` | VARCHAR(500) | 사유/메모 |
| `appr_req_user_id` | sy_user.user_id | 요청자 |
| `appr_req_date` | TIMESTAMP | 요청일시 |
| `appr_aprv_user_id` | sy_user.user_id | 결재자 (승인 처리자) |
| `appr_aprv_date` | TIMESTAMP | 결재일시 |

## 8. 배송비 부담 결정 흐름도

```
클레임 사유 확인
  │
  ├─ 판매자 귀책 (DEFECT / DAMAGE / WRONG_ITEM / SIZE_COLOR_ERROR)
  │    → 배송비 전액 판매자 부담
  │    → 추가결제 요청 없음
  │
  ├─ 고객 귀책 (CHANGE_OF_MIND)
  │    → 고객 배송비 부담
  │    │
  │    ├─ 환불금액에서 차감 가능한 경우
  │    │    → 환불금액에서 자동 차감 (추가결제 요청 불필요)
  │    │
  │    └─ 환불금액보다 배송비가 클 경우
  │         → 추가결제 요청 생성 (appr_target_cd = DLIV)
  │
  └─ 협의 (OTHER)
       → 관리자 판단으로 추가결제 요청 여부 결정
```
