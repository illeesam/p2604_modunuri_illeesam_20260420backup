# 711. 정산 마감 정책

## 목적
판매자 정산액 계산 및 월별 정산 마감 정책 정의

## 범위
- 정산기간 관리
- 정산대상 선정
- 정산액 계산
- 마감처리

## 정산 주기
- **기본**: 월 1회 (매월 마지막 영업일)
- **정산기간**: 1일 ~ 말일 (한국시간 기준)
- **마감시점**: 매월 마지막 영업일 자정

## 주요 정책

### 1. 정산대상
- **포함**:
  - 배송완료 상태인 주문
  - 반품/교환 완료된 주문
  - 취소 최종 처리된 주문
- **제외**:
  - PENDING, PREPARING 상태 주문
  - 반품 진행 중인 주문
  - 판매자 정지 기간 주문

### 2. 정산액 계산
```
정산액 = 총주문금액 - 총환불금액 - 총할인금액 - 수수료 + 조정금액 + 기타조정금액

총주문금액(total_order_amt) = 당월 신규 주문 귀속분 합계
총환불금액(total_return_amt) = 당월 환불 확정분 합계 (타월 주문 환불 포함)
총할인금액(total_discnt_amt) = 쿠폰/프로모션 할인 합계
수수료(commission_amt)       = 정산기준 수수료율 × 정산대상금액
```

**od_pay 입출금 구조와 정산 집계 기준**

`od_pay` 한 레코드에 입금·출금이 공존하며 명시적 입출금 구분 컬럼이 없다.
정산 집계 시 아래 기준으로 방향을 구분한다.

| 집계 대상 | od_pay 컬럼 | 조건 | 정산 반영 |
|---|---|---|---|
| 입금 (판매 수취) | `pay_amt` | `pay_status_cd = COMPLT` | `total_order_amt` (+) |
| 출금 (환불 지급) | `refund_amt` | `refund_status_cd = COMPLT` | `total_return_amt` (-) |
| 교환 추가결제 | 신규 `od_pay` 레코드 | `pay_status_cd = COMPLT` | `total_order_amt` (+) |

- `BANK_TRANSFER` 입금: 관리자 수동 확인 후 COMPLT → 수동 확인 시점이 집계 기준일
- `VBANK` 입금: `vbank_deposit_date` 기록 시점이 집계 기준일. 부족입금 주의 (pg_response 파싱 검증 필요)
- `TOSS·KAKAO·NAVER·MOBILE`: PG 승인 시점 기준 자동 집계

### 3. 타월(他月) 주문 환불 처리 원칙
- **기준**: 환불은 **환불 확정 시점의 귀속 월**에 반영 (발생주의 원칙)
- **예시**: 1월 주문 → 3월 환불 확정 → **3월 정산에서 차감**
- **1월 정산**: 이미 `CLOSED`/`PAID` 상태 → 재오픈 없음
- **3월 정산**: `st_settle_raw`에 `raw_type_cd=CLAIM`으로 음수(-) 원장 수집 → `total_return_amt`에 누적
- **마이너스 정산**: `final_settle_amt < 0` 발생 시 → 다음 달 이월(`st_settle_adj`) 또는 수동 조정

### 3. 정산액 조정
- **배송료정산**: 
  - 고객부담 배송료 회수
  - 판매자부담 배송료 공제
- **반품환불**:
  - 전액반품: 판매액 100% 환불
  - 부분반품: 해당상품금액만 환불
- **할인공제**:
  - 쿠폰할인: 100% 공제 (판매자 부담)
  - 카테고리할인: 50% 공제 (플랫폼/판매자 분담)

### 4. 정산 보류
- **보류사유**:
  - 환불 진행 중 (+7일)
  - 거래 분쟁 (+30일)
  - 판매자 정지 (해제까지)
  - 정산계좌 변경 대기 (확인까지)
- **보류해제**: 사유 제거 후 다음 정산기에 포함

### 5. 정산 세금
- **수익세**: 10% (사업자 등록 여부별)
  - 사업자: 사업자등록증 기준 세금처리
  - 일반인: 10% 원천징수
- **부가세**: 별도 계산 (사업자만 해당)

### 6. 정산 결과 조회
- **정산명세**: PDF 형식 다운로드 가능
  - 기간별 판매액
  - 수수료 계산내역
  - 환불/조정사항
  - 최종정산액
- **정산이력**: 월별 정산내역 조회

### 7. 정산 지연
- **사유**:
  - 계정 미승인 (-1주)
  - 서류 미제출 (-2주)
  - 거래 분쟁 (-1개월)
- **통지**: 사전 공지 (7일 전)

## 정산액 계산 예시
```
주문기간: 2026년 4월
주문금액: 100,000원

1. 판매액 계산
   상품금액: 100,000원
   배송료: 2,500원 (고객부담)
   판매액: 100,000원

2. 수수료 계산 (수수료율 5%)
   수수료: 100,000 × 5% = 5,000원

3. 배송료 처리
   고객배송료: 2,500원 (회수)
   플랫폼배송료: 2,000원 (기본요금)
   배송비순이익: 500원

4. 최종정산액
   정산액 = 100,000 - 5,000 + 500 = 95,500원

5. 세금공제 (사업자가 아닌 경우)
   세금 10%: 95,500 × 10% = 9,550원
   최종지급액: 95,500 - 9,550 = 85,950원
```

## 주요 필드 (st_settle)
| 컬럼 | 설명 | 규칙 |
|------|------|------|
| settle_id | 정산ID | YYMMDDhhmmss+rand4 |
| vendor_id | 업체ID | FK sy_vendor |
| settle_ym | 정산년월 | YYYYMM |
| total_order_amt | 총주문금액 | 당월 신규 주문 귀속분 |
| total_return_amt | 총환불금액 | 당월 환불 확정분 (타월 주문 환불 포함) |
| total_claim_cnt | 환불 건수 | raw_type_cd=CLAIM 집계 |
| total_discnt_amt | 총할인금액 | 쿠폰/프로모션 합계 |
| commission_rate | 수수료율 | % (st_settle_config 기준) |
| commission_amt | 수수료금액 | settle_target_amt × rate |
| settle_amt | 기본정산금액 | order - return - discnt - commission |
| adj_amt | 조정금액 | st_settle_adj 합계 |
| etc_adj_amt | 기타조정금액 | st_settle_etc_adj 합계 |
| final_settle_amt | 최종정산금액 | settle + adj + etc_adj |
| settle_status_cd | 상태 | DRAFT/CONFIRMED/CLOSED/PAID |

## 관련 테이블
| 테이블명 | 한글설명 |
|---------|---------|
| st_settle_config | 정산기준 설정 (업체/카테고리별 수수료율, 정산주기) |
| st_settle_raw | 정산수집원장 (od_order_item 기반 원천 수집) |
| st_settle | 정산 마스터 (업체별 월정산 집계) |
| st_settle_item | 정산 항목 (주문항목별 명세) |
| st_settle_adj | 정산 조정 (수동 가감) |
| st_settle_etc_adj | 정산 기타조정 (배송비/위약금 등) |
| st_settle_close | 정산 마감 이력 |

**SETTLE_STATUS**: `DRAFT`(작성중) / `CONFIRMED`(확정) / `CLOSED`(마감) / `PAID`(지급완료)

**정산기준 우선순위**: `vendor_id + category_id` > `vendor_id` > `site_id`(전체기준)

## 제약사항
- 정산은 월 1회만 진행
- 정산중 거래 분쟁 발생 시 정산 보류
- 미정산액은 다음달에 이월
