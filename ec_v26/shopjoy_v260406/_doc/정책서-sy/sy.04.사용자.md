# 912. 사용자 관리 정책

## 목적
관리자 및 운영자 계정 관리 정책 정의

## 범위
- 사용자 등록 및 인증
- 계정 상태 관리
- 로그인 및 세션 관리
- 비밀번호 정책

## 사용자 상태 (USER_STATUS)
| 상태 | 코드 | 설명 |
|------|------|------|
| 활성 | ACTIVE | 정상 사용 가능 |
| 휴면 | DORMANT | 장기미접속 |
| 정지 | SUSPENDED | 정책위반 정지 |
| 삭제 | DELETED | 퇴직 또는 삭제 |

## 주요 정책

### 1. 사용자 등록
- **필수항목**:
  - 로그인ID (영문/숫자, UNIQUE)
  - 비밀번호 (8자 이상, 특수문자 포함)
  - 이름
  - 이메일
  - 소속부서
- **역할**: 등록 시점에 권한 할당
- **초기상태**: ACTIVE

### 2. 비밀번호 정책
- **최소길이**: 8자
- **필수조건**:
  - 영문 대문자 1개 이상
  - 영문 소문자 1개 이상
  - 숫자 1개 이상
  - 특수문자 1개 이상
- **변경주기**: 90일마다 변경 권장
- **이력관리**: 최근 3개 암호 재사용 불가
- **암호화**: bcrypt 암호화 저장

### 3. 로그인 관리
- **시도제한**: 실패 5회 후 15분 정지
- **로그기록**: 모든 로그인 시도 기록
  - 성공/실패 여부
  - IP 주소
  - 접속시간
- **세션관리**:
  - 기본 타임아웃: 30분
  - 최대세션: 3개 (다중로그인 허용)
  - 자동로그아웃: 비활성 30분

### 4. 사용자 상태 변경
- **ACTIVE → DORMANT**: 90일 미접속 자동
- **SUSPENDED**: 관리자 수동 정지
  - 사유: 정책위반, 부정행위 등
  - 기간: 1주 ~ 1개월
- **DELETED**: 퇴직 시 논리삭제
  - 계정 비활성화
  - 데이터 보관 (규정기간)

### 5. 비밀번호 재설정
- **초기화**: 관리자 요청 시 임시 비밀번호 발급
- **이메일 확인**: 등록 이메일로 재설정 링크 발송
- **유효기간**: 24시간
- **재시도**: 최대 3회

### 6. 접근 제어
- **IP 제한**: 특정 IP만 접근 가능 (선택)
- **시간제한**: 접근 가능 시간대 설정 (선택)
- **다중인증**: 관리자급 2FA 권장
- **인증서**: VPN 접근 시 인증서 필수

### 7. 감시 및 감사
- **로그인로그**: 모든 시도 기록
- **활동로그**: 중요 작업 기록
- **비정상탐지**: 
  - 비정상 IP 접근
  - 야간 대량작업
  - 대량 데이터 다운로드

## 주요 필드
| 필드 | 설명 | 규칙 |
|------|------|------|
| user_id | 사용자ID | YYMMDDhhmmss+rand4 |
| login_id | 로그인ID | 영문/숫자, UNIQUE |
| user_password | 비밀번호 | bcrypt 암호화 |
| user_nm | 사용자명 | 필수 |
| user_email | 이메일 | 필수 |
| dept_id | 부서ID | FK sy_dept |
| user_status_cd | 상태 | USER_STATUS 코드 |
| last_login | 최근로그인 | TIMESTAMP |
| login_fail_cnt | 실패횟수 | INTEGER |

## 관련 테이블
- sy_user: 사용자 기본 정보
- sy_user_login_log: 로그인 이력
- sy_user_token_log: 토큰/세션 관리

## 제약사항
- 로그인ID 변경 불가능
- 비밀번호 90일마다 변경 권장
- 실패 5회 후 자동 정지
