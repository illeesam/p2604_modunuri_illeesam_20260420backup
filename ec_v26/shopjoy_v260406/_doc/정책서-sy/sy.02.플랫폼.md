# 111. 플랫폼 기본 정책

## 목적
멀티사이트 플랫폼 운영을 위한 기본 정책 및 규칙 정의

## 범위
- 플랫폼 전체 사이트 관리
- 기본 설정 및 운영 정책
- 플랫폼 수준의 코드/분류 관리

## 주요 정책

### 1. 사이트 관리
- **정의**: 플랫폼 내 독립적인 쇼핑몰 운영 단위
- **사이트 상태**: ACTIVE (활성), INACTIVE (비활성)
- **필수 정보**:
  - 사이트명 (site_nm)
  - 도메인 (site_domain)
  - 업체명 (site_ceo)
  - 사업자번호 (site_business_no)
  - 연락처 (site_phone, site_email)
  - 주소 (site_zip_code, site_address)

### 2. 테마 전시
- **사이트별 독립 테마**: 01, 02, 03 등 여러 테마 운영 가능
- **테마 구성**:
  - 베이지/카키톤 (01)
  - 민트/세이지 그린 톤 (02)
  - 로얄 퍼플 톤 (03)

### 3. 운영 권한
- **사이트별 독립 운영**: 각 사이트는 독립적인 관리자/판매자 구성
- **권한 관리**: sy_role, sy_user를 통해 세분화된 접근 제어

### 4. 데이터 격리
- **site_id 기반 데이터 분리**: 모든 거래/사용자 정보는 site_id로 격리
- **보안**: 교차 사이트 접근 불가

## 관련 테이블
- sy_site: 사이트 기본 정보
- sy_user: 운영자 계정
- sy_role: 권한 정의
- sy_dept: 조직 구조
