# 914. 권한 관리 정책

## 목적
역할 기반 접근 제어(RBAC) 및 권한 관리 정책 정의

## 범위
- 역할(Role) 정의
- 권한(Permission) 관리
- 역할-권한 할당
- 사용자-역할 할당

## 기본 역할 (Role)
| 역할 | 코드 | 권한범위 |
|------|------|---------|
| 슈퍼관리자 | SUPER_ADMIN | 전체 시스템 |
| 관리자 | ADMIN | 회원/상품/주문/배송/정산 |
| 운영자 | OPERATOR | 주문/배송/고객센터 |
| 상품담당자 | PRODUCT_MANAGER | 상품/카테고리 |
| 판매담당자 | SALES_MANAGER | 주문/배송 |
| 마케팅담당자 | MARKETING_MANAGER | 쿠폰/이벤트/전시 |
| 정산담당자 | SETTLE_MANAGER | 정산/수수료 |
| 고객서비스 | CUSTOMER_SERVICE | 클레임/채팅 |

## 주요 정책

### 1. 역할 정의
- **역할명**: 40자 이내
- **설명**: 150자 이내
- **권한범위**: 다음 페이지/기능 권한 상세 정의
- **부서**: 특정 부서 할당 (선택)

### 2. 권한 구조
```
권한 = 메뉴접근 + 기능실행

기능실행:
├─ CREATE (생성)
├─ READ (조회)
├─ UPDATE (수정)
└─ DELETE (삭제)
```

### 3. 역할별 권한 예시
#### 3-1. 슈퍼관리자 (SUPER_ADMIN)
- 모든 메뉴 접근
- 모든 기능 실행
- 사용자/역할 관리 가능
- 시스템설정 변경 가능

#### 3-2. 관리자 (ADMIN)
- 회원/상품/주문/배송 관리
- 클레임/환불 처리
- 정산 조회만 가능
- 시스템설정 불가

#### 3-3. 운영자 (OPERATOR)
- 주문/배송 조회 및 수정
- 클레임 처리
- 회원 조회만 가능
- 상품 조회만 가능

### 4. 권한 할당
- **역할→권한**: 관리자가 역할에 권한 할당
- **사용자→역할**: 부서장이 사용자에 역할 할당
- **권한상속**: 하위역할이 상위역할 권한 일부 가능

### 5. 권한 캐싱
- **캐시주기**: 5분
- **변경시**: 즉시 반영 (캐시 초기화)
- **성능**: 매요청시 권한 검증 (캐시로 빠름)

### 6. 권한 감시
- **로그기록**: 모든 권한변경 기록
- **비정상탐지**:
  - 권한없는 기능 실행
  - 대량 데이터 접근
  - 야간 대량 작업
- **알림**: 비정상 시도 자동 알림

### 7. 권한 회수
- **자동회수**:
  - 사용자 삭제 시
  - 부서 폐지 시
  - 역할 삭제 시
- **수동회수**: 관리자 직접 회수

### 8. 위임권한
- **한시적**: 특정기간만 권한 위임
- **승인**: 상급자 승인 필수
- **추적**: 위임자/위임기간 기록

## 권한 매핑 예시
```
사용자A
├─ 역할: 주문담당자
│  └─ 메뉴: 주문관리
│     └─ 권한:
│        ├─ READ (조회)
│        ├─ UPDATE (상태변경)
│        └─ DELETE (취소/환불 불가)
│
└─ 특수권한: 배송 수동입금 (한시적)
   └─ 기간: 2026-04-16 ~ 2026-05-16
```

## 주요 필드
| 필드 | 설명 | 규칙 |
|------|------|------|
| role_id | 역할ID | YYMMDDhhmmss+rand4 |
| role_nm | 역할명 | 40자 이내 필수 |
| role_desc | 설명 | 150자 이내 |
| role_type_cd | 유형 | SYSTEM/CUSTOM |
| use_yn | 사용여부 | Y/N |

## 관련 테이블
- sy_role: 역할 정보
- sy_role_menu: 역할-메뉴 매핑
- sy_user_role: 사용자-역할 매핑

## 제약사항
- 기본역할은 변경 불가능
- 권한은 더 큰 권한으로만 상향 가능
- 역할 삭제 전 사용자 재할당 필수

## 변경이력
- 2026-04-16: 초기 작성
