# 913. 메뉴 관리 정책

## 목적
관리자 시스템 메뉴 구조 및 접근 제어 정책 정의

## 범위
- 메뉴 계층 구조
- 메뉴 권한 관리
- 메뉴 노출 조건
- 메뉴 순서 관리

## 메뉴 구조
```
메인메뉴 (깊이 1)
└─ 서브메뉴 (깊이 2)
   └─ 세부기능 (깊이 3)
```

## 주요 정책

### 1. 메뉴 생성
- **필수항목**:
  - 메뉴명
  - 메뉴유형 (GROUP/PAGE/FUNCTION)
  - 접근URL 또는 기능ID
  - 정렬순서
- **선택항목**:
  - 아이콘
  - 설명
  - 열기방식 (새창/현재창)

### 2. 메뉴 유형
| 유형 | 코드 | 설명 |
|------|------|------|
| 그룹 | GROUP | 메뉴그룹 (클릭불가) |
| 페이지 | PAGE | 페이지 링크 |
| 기능 | FUNCTION | 개별 기능 |

### 3. 메뉴 계층
- **깊이1**: 메인메뉴 (최대 10개)
  - 회원관리, 상품관리, 주문관리 등
- **깊이2**: 서브메뉴 (메인당 최대 20개)
  - 회원목록, 회원상세 등
- **깊이3**: 세부기능 (서브당 최대 10개)
  - 회원등록, 회원수정, 회원삭제

### 4. 권한과 연결
- **메뉴권한**: 각 메뉴별로 필요 권한 지정
- **역할권한**: 역할에 메뉴권한 할당
- **사용자권한**: 사용자 → 역할 → 메뉴 순으로 결정

### 5. 메뉴 노출 조건
- **권한**: 필수권한 없으면 미노출
- **상태**: 비활성 메뉴는 미노출
- **부서**: 부서별 메뉴 제한 (선택)
- **시간**: 특정시간대만 노출 (선택)

### 6. 메뉴 정렬
- **정렬필드**: sort_ord (정수)
- **정렬방식**: 오름차순 (작은값이 상위)
- **범위**: 깊이별로 독립 정렬
  - 메인: 0~999
  - 서브: 0~999 (메인당)

### 7. 메뉴 변경
- **생성**: 즉시 반영
- **수정**: 즉시 반영
- **삭제**: 비활성화만 가능
  - 실제 삭제는 미사용 1개월 후

### 8. 메뉴 감시
- **로그기록**: 메뉴접근 기록
- **비정상감지**:
  - 권한없는 메뉴 접근
  - 숨겨진 메뉴 직접접근
  - 비활성 메뉴 접근

## 주요 필드
| 필드 | 설명 | 규칙 |
|------|------|------|
| menu_id | 메뉴ID | YYMMDDhhmmss+rand4 |
| parent_menu_id | 상위메뉴ID | NULL이면 메인 |
| menu_nm | 메뉴명 | 50자 이내 |
| menu_type_cd | 유형 | GROUP/PAGE/FUNCTION |
| menu_url | 접근URL | 절대경로 |
| menu_function_id | 기능ID | 기능-기반 구현시 |
| sort_ord | 정렬순서 | INTEGER |
| menu_icon | 아이콘 | FontAwesome 클래스 |
| menu_desc | 설명 | TEXT |
| use_yn | 사용여부 | Y/N |
| role_id | 필요권한 | FK sy_role |

## 관련 테이블
- sy_menu: 메뉴 정보
- sy_role_menu: 역할-메뉴 매핑
- sy_api_log: 메뉴 접근 로그

## 제약사항
- 순환참조 방지 (자신을 상위로 설정 불가)
- 깊이 3 초과 불가
- 메뉴 삭제는 비활성화만 가능
