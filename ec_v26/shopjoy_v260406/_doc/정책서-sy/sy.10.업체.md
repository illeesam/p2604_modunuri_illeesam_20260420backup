# 916. 업체(벤더) 기본정보 정책

## 목적
플랫폼 내 판매자(업체/벤더) 기본정보 관리 정책

## 범위
- 업체 등록 및 변경
- 업체정보 관리
- 담당자 관리
- 계약정보 관리

## 주요 정책

### 1. 업체 등록 정보
#### 1-1. 기본정보
- **업체명**: 50자 이내 (UNIQUE)
- **대표자명**: 필수
- **사업자번호**: UNIQUE (개인용사업자번호 구분)
- **업종**: INDUSTRY_CODE 선택
- **설립일**: DATE

#### 1-2. 연락처
- **대표번호**: 필수
- **팩스**: 선택
- **이메일**: 필수
- **주소**: 사업장주소 필수
  - 우편번호
  - 기본주소
  - 상세주소

#### 1-3. 통장정보
- **은행명**: 필수
- **계좌번호**: 필수 (암호화 저장)
- **예금주명**: 필수
- **검증**: 계좌 정상 여부 확인

#### 1-4. 계약정보
- **계약일**: 업체 등록일
- **수수료율**: 기본 5% (변경 가능)
- **계약기간**: 1년 (자동연장)
- **계약담당자**: 플랫폼 담당자 지정

### 2. 담당자 관리
- **최대인원**: 5명
- **역할**:
  - 대표담당자 (1명, 필수)
  - 부담당자 (4명 이내)
- **권한**: 상품관리, 주문/배송, 정산조회

### 3. 정산계좌 관리
- **변경가능**: 월 1회 (신청 후 확인)
- **검증**: 계좌 본인 검증 필수
  - 송금검증: 소액 송금 후 확인
  - 계좌확인: 계좌주 동의 필수
- **변경기간**: 1개월

### 4. 사업자정보 확인
- **서류제출**:
  - 사업자등록증 (필수)
  - 통장사본 (필수)
  - 대표자신분증 (필수)
- **유효성검사**:
  - 국세청 조회 (자동)
  - 통관 확인 (부가세)
  - 폐업여부 확인

### 5. 계약내용
- **수수료**: 기본 5%, 변경 가능
  - 카테고리별 차등 가능
  - 분기별 1회 변경 신청 가능
- **배송료**: 판매자 설정 또는 플랫폼 기본
- **환불정책**: 상품별 설정

### 6. 정산정책
- **정산주기**: 월 1회
- **정산기간**: 1일~말일
- **지급기한**: 마감 후 5 영업일
- **수수료공제**: 자동 공제

### 7. 계약해지
- **해지사유**:
  - 판매자 요청 (1개월 공지)
  - 계약조건 위반 (즉시)
  - 폐업 (증명서 제출)
- **절차**:
  1. 정산완료 확인
  2. 보증금 반환
  3. 계정 폐쇄

### 8. 정보 보안
- **암호화**: 계좌번호, 신분증 암호화
- **접근제한**: 해당 담당자와 관리자만 열람
- **감시**: 정보 조회/변경 기록

## 주요 필드
| 필드 | 설명 | 규칙 |
|------|------|------|
| vendor_id | 업체ID | YYMMDDhhmmss+rand4 |
| vendor_nm | 업체명 | 50자, UNIQUE |
| ceo_nm | 대표자명 | 필수 |
| business_no | 사업자번호 | UNIQUE, 암호화 |
| phone | 연락처 | 필수 |
| email | 이메일 | 필수 |
| bank_account | 정산계좌 | 암호화 저장 |
| fee_rate | 수수료율 | DECIMAL 기본 5% |
| contract_date | 계약일 | TIMESTAMP |
| contract_end | 계약종료일 | TIMESTAMP |
| vendor_status_cd | 상태 | PENDING/APPROVED/SUSPENDED/CLOSED |

## 관련 테이블
- sy_vendor: 업체 기본 정보
- sy_vendor_manager: 업체 담당자
- sy_vendor_settle: 업체 정산

## 제약사항
- 사업자번호 중복 불가
- 정산계좌 월 1회만 변경 가능
- 폐업 업체는 상품판매 중지
