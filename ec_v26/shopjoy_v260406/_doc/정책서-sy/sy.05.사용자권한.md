# sy.05. 사용자 권한 관리 정책

## 목적
관리자 사용자(`sy_user`)에 대한 역할 할당 및 권한 적용 정책 정의.
역할 정의(sy.07.권한.md)와 구분하여, **사용자-역할 매핑 운영** 절차와 적용 기준을 다룬다.

## 범위
- 사용자-역할 할당 및 회수 (`sy_user_role`)
- 역할-메뉴 권한 매핑 (`sy_role_menu`)
- 권한 캐싱 및 즉시 반영
- 권한 위임 및 한시적 부여
- 감사 및 이력 관리

## 관련 테이블
| 테이블명 | 설명 |
|---------|------|
| sy_user | 관리자 사용자 |
| sy_user_role | 사용자-역할 매핑 |
| sy_role | 역할 마스터 (역할 정의 → sy.07.권한.md 참조) |
| sy_role_menu | 역할-메뉴 권한 매핑 |
| sy_menu | 메뉴 마스터 |

## 주요 정책

### 1. 사용자-역할 할당 (`sy_user_role`)
- **할당 주체**: 부서장 이상 또는 SUPER_ADMIN·ADMIN 역할 보유자
- **할당 단위**: 사용자 1명 + 역할 1개 (다중 역할 허용, `sy_user_role` 레코드 복수 등록)
- **할당 기준**:
  - 소속 부서(`dept_id`) 업무 범위 내 역할만 할당 권장
  - 자신보다 상위 역할 할당 불가 (ADMIN이 SUPER_ADMIN 할당 불가)
- **즉시 반영**: 할당 즉시 권한 캐시 초기화 → 다음 요청부터 적용

### 2. 역할-메뉴 권한 매핑 (`sy_role_menu`)
- **관리 주체**: SUPER_ADMIN만 수정 가능
- **권한 구조**: 역할 → 메뉴 접근 허용 여부 + 기능 실행 권한 (READ/CREATE/UPDATE/DELETE)
- **변경 시**: 해당 역할 보유 전 사용자 캐시 즉시 초기화

### 3. 권한 캐싱
- **캐시 유지**: 5분
- **즉시 초기화 트리거**:
  - 사용자-역할 할당·회수
  - 역할-메뉴 권한 변경
  - 사용자 상태 변경 (SUSPENDED 등)
- **성능**: 매 요청마다 DB 조회 없이 캐시에서 권한 검증

### 4. 권한 회수
```
자동 회수:
  - 사용자 삭제(DELETED) → 전 역할 자동 회수
  - 부서 폐지 → 해당 부서 사용자 역할 재검토 필요 (자동 회수 아님)
  - 역할 삭제 → sy_user_role에서 해당 role_id 레코드 삭제

수동 회수:
  - 담당자 변경, 업무 이관, 정책 위반 등
  - ADMIN 이상 권한 보유자가 직접 회수
```

### 5. 한시적 권한 위임
- **목적**: 휴가·출장 등 임시 업무 대행
- **조건**: 위임자의 상급자 승인 필수
- **기록**:
  - 위임자(`delegator_user_id`), 수임자(`delegatee_user_id`)
  - 위임 기간 (`delegate_start_date` ~ `delegate_end_date`)
  - 위임 사유
- **만료**: `delegate_end_date` 도래 시 자동 회수

### 6. 복수 역할 충돌 처리
- 한 사용자가 여러 역할 보유 시 **합집합(Union)** 원칙 적용
  - 어느 역할에서라도 허용된 기능이면 실행 가능
- 예외: 역할 간 명시적 충돌 정의 시 상위 역할 우선

### 7. 권한 검증 흐름
```
사용자 요청
  → 로그인 세션에서 user_id 확인
  → 캐시에서 해당 user_id의 역할 목록 조회
  → 역할-메뉴 권한 매핑에서 요청 기능 허용 여부 확인
  → 허용: 기능 실행 / 거부: 401/403 응답
```

### 8. 감사 및 이력
- 역할 할당·회수 이력: `sy_user_role` 변경 기록 (`reg_by`, `reg_date`)
- 역할-메뉴 권한 변경 이력: `sy_role_menu` 변경 기록
- 권한 없는 기능 접근 시도: `sy_api_log`에 403 기록

## 제약사항
- 자신보다 상위 역할은 할당 불가
- `SUPER_ADMIN` 역할 할당은 `SUPER_ADMIN`만 가능
- 역할-메뉴 권한 수정은 `SUPER_ADMIN`만 가능
- 한시적 위임은 상급자 승인 기록 필수
