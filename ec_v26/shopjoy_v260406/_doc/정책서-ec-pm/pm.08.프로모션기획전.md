# 517. 프로모션 기획전 정책

## 목적
시즌·브랜드·테마·협업 등 기획전을 생성하고, 대상 상품 진열 및 공개 상태를 관리하는 정책 정의

## 범위
- 관련 역할: 관리자(기획전 등록·상품 배치·공개 관리), 회원(기획전 페이지 탐색)
- 관련 시스템: 프로모션 관리, 상품 전시, 사용자 페이스

## 주요 정책

### 1. 기획전 생성 및 공개 흐름
```
DRAFT(초안 저장) → ACTIVE(공개) → ENDED(종료)
```
- 관리자가 기획전 기본정보·썸네일·배너 등록 후 저장 → `plan_status_cd: DRAFT`
- 공개 처리 시 `ACTIVE`, 사용자 페이지에서 탐색 가능
- `end_date` 경과 또는 관리자 수동 종료 시 `ENDED` 전환
- `ENDED` 기획전은 읽기 전용 (재활성화 불가, 복사 후 신규 생성 권장)

### 2. 기획전 유형 (plan_type_cd)
- **SEASON (시즌)**: 봄·여름·가을·겨울 등 계절 테마 기획전 (예: 2026 여름 컬렉션)
- **BRAND (브랜드)**: 특정 브랜드 집중 기획전 (예: OO 브랜드 특가)
- **THEME (테마)**: 선물·캠핑·홈카페 등 라이프스타일 테마
- **COLLAB (협업)**: 타 브랜드·인플루언서 콜라보 기획전

### 3. 기획전 상품 등록 (pm_plan_item)
- 기획전 1개에 상품 N개 등록 가능 (상한 제한 없음)
- `sort_ord` 필드로 진열 순서 수동 관리 (숫자 낮을수록 앞에 노출)
- `plan_item_memo`: 해당 상품에 대한 기획전 전용 설명 (예: "이번 시즌 신상", "한정수량")
- 동일 상품을 복수 기획전에 등록 가능 (상품 중복 허용)

### 4. 노출 및 정렬
- 기획전 목록 페이지: `start_date` 최신순 또는 관리자 설정 순서 기준
- 기획전 내 상품: `sort_ord` ASC 기준 노출
- `thumbnail_url`: 목록 카드형 섬네일
- `banner_url`: 기획전 상세 상단 배너 이미지

### 5. 기획전 기간 관리
- `start_date` / `end_date`: 기획전 노출 기간 (DATE 타입)
- `start_date` 이전: 미노출 (`DRAFT` 또는 `ACTIVE` 상관없이 사용자에게 비표시 가능)
- `end_date` 경과 후 배치 실행 시 `ACTIVE → ENDED` 자동 전환

### 6. 기획전과 할인·쿠폰 연계
- 기획전 자체는 가격 할인 기능을 갖지 않음
- 연계 할인은 `pm_discnt` (할인정책, PRODUCT 대상)로 별도 등록
- 기획전 상품에 쿠폰 적용 가능 (쿠폰 정책 참조)

## 상태 코드

### PLAN_STATUS (기획전 상태)
| 코드그룹 | 코드값 | 라벨 | 설명 |
|---------|--------|------|------|
| PLAN_STATUS | DRAFT | 초안 | 저장됨, 사용자에게 미노출 |
| PLAN_STATUS | ACTIVE | 공개 | 사용자 노출, 탐색 가능 |
| PLAN_STATUS | ENDED | 종료 | 기간 종료 또는 수동 종료, 읽기 전용 |

### PLAN_TYPE (기획전 유형)
| 코드그룹 | 코드값 | 라벨 | 설명 |
|---------|--------|------|------|
| PLAN_TYPE | SEASON | 시즌 | 계절 테마 기획전 |
| PLAN_TYPE | BRAND | 브랜드 | 특정 브랜드 집중 기획전 |
| PLAN_TYPE | THEME | 테마 | 라이프스타일 주제 기획전 |
| PLAN_TYPE | COLLAB | 협업 | 콜라보레이션 기획전 |

## 관련 테이블
| 테이블명 | 한글설명 |
|---------|---------|
| pm_plan | 기획전 마스터 (유형·제목·배너·기간·상태) |
| pm_plan_item | 기획전 상품 목록 (상품 매핑·진열순서·메모) |

## 제약사항
- `ENDED` 기획전은 재활성화 불가 (ACTIVE 전환 차단)
- `start_date > end_date` 저장 불가
- `pm_plan_item.sort_ord` 동일값 허용, 동일 시 등록 순 처리
- `thumbnail_url` 미입력 시 기본 이미지 표시
- `DRAFT` 상태에서도 상품 등록 가능 (사전 작업 허용)
