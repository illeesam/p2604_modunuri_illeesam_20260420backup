# 사용자(Front Office) UX 정책

## 1. 화면 프레임 구조

```
┌─────────────────────────────────────────────────────────────┐
│  [AppHeader]  로고 | 사이트배지 | GNB | 장바구니 | 로그인    │  ← 고정
├─────────────────────────────────────────────────────────────┤
│  [사이트 리본]  🌿 Mint Edition / 👑 Luxe Edition (02·03만) │  ← 조건부
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  [페이지 콘텐츠]   해시 라우팅으로 동적 교체                │
│                                                              │
│  ┌──────────────┐  ┌──────────────────────────────────────┐ │
│  │ [MyLayout    │  │  콘텐츠 영역                         │ │
│  │  좌측메뉴]   │  │  (마이페이지 전용 2컬럼 레이아웃)    │ │
│  │  (마이페이지 │  │                                      │ │
│  │   전용)      │  │                                      │ │
│  └──────────────┘  └──────────────────────────────────────┘ │
│                                                              │
├──────────────────────────────────────────────────────────────┤
│  [AppFooter]  회사정보 | 바로가기 레이어 모달               │
└─────────────────────────────────────────────────────────────┘
```

- **상단바(AppHeader)**: 전체 페이지 공통 고정
- **좌측 사이드바**: 마이페이지(MyLayout) 전용. 그 외 일반 페이지는 full-width 단일 컬럼
- **우측 패널**: 없음 (사용자 페이스는 우측 패널 없음)
- **푸터(AppFooter)**: 전체 페이지 공통

## 2. 상단바 (AppHeader)

### 2.1 구성 요소 배치

```
[로고]  [사이트배지]  [GNB 메뉴 ···]          [🛒 N]  [👤]
                                              장바구니  로그인/마이
```

| 요소 | 설명 |
|---|---|
| 로고 | 클릭 시 `home` 이동 |
| 사이트 배지 | `01` / `02` / `03` — hover 시 테마명 툴팁 표시 |
| GNB 메뉴 | `window.SITE_CONFIG.menus` 기반 카테고리 링크 |
| 장바구니 아이콘 | 장바구니 담긴 수량 뱃지 표시. 클릭 시 `cart` 페이지 이동 |
| 로그인/마이페이지 | 미로그인: 👤 아이콘 → 로그인 페이지(`login`). 로그인 후: 사용자명 또는 마이페이지 아이콘 |

### 2.2 로그인 방식

| 방식 | 설명 |
|---|---|
| 이메일/비밀번호 | 기본 로그인. `api/base/users.json` 목업 기준 |
| 카카오 로그인 | 소셜 로그인 (OAuth 연동) |
| 네이버 로그인 | 소셜 로그인 (OAuth 연동) |

로그인 처리 흐름:
```
로그인 폼 입력 → frontAuthStore.login(email, pw)
  → API 호출 → 토큰 발급 → localStorage 저장
  → frontAuth.state.isLoggedIn = true
  → 이전 페이지 또는 home으로 이동
```

### 2.3 로그인 후 표시 정보

| 표시 위치 | 내용 |
|---|---|
| 헤더 우측 아이콘 | 사용자명 또는 마이페이지 아이콘으로 전환 |
| hover 드롭다운 | 사용자명, 회원 등급 배지, 마이페이지 링크, 로그아웃 버튼 |
| 장바구니 뱃지 | 담긴 상품 수량 실시간 반영 |

### 2.4 로그인 상태 유지

```js
// localStorage 토큰 자동 복원
// base/frontAuth.js
window.frontAuth = {
  state: Vue.reactive({ isLoggedIn: false, user: null }),
  async init() { /* localStorage 토큰 확인 → 자동 복원 */ },
  async logout() { /* 토큰 제거 + state 초기화 */ },
};

// Pinia 스토어 (base/stores/frontAuthStore.js)
frontAuthStore: { isLoggedIn, user, token }
```

### 2.5 모바일 대응

- 햄버거 메뉴(≡) 버튼 → 전체화면 슬라이드 메뉴 (CSS transition)
- GNB 메뉴 축소 → 햄버거 내 전체 메뉴 표시

## 3. 사이트 테마 (FRONT_SITE_NO)

| FRONT_SITE_NO | 테마 | 특징 |
|---|---|---|
| `01` | 기본 (베이지/카키) | 기준 테마. 리본 없음 |
| `02` | Mint Edition (민트/세이지) | 상단 🌿 리본 표시 |
| `03` | Luxe Edition (로얄 퍼플) | 상단 👑 리본 표시 |

- URL 오버라이드: `?FRONT_SITE_NO=02` → localStorage 저장 후 쿼리 자동 제거
- CSS 변수로 테마 전환: `--accent`, `--text-primary`, `--bg-base`, `--bg-card`, `--border`, `--shadow`

## 4. 좌측 사이드바 — 마이페이지 전용 (MyLayout)

### 4.1 표시 조건

마이페이지 계열 페이지(`myOrder`, `myClaim`, `myCoupon`, `myCache`, `myContact`, `myChatt`)에서만 표시.  
일반 페이지는 full-width 단일 컬럼.

### 4.2 레이아웃

```
lg 이상 (≥1024px):
  ┌─────────────┬──────────────────────────────┐
  │  좌측 메뉴  │  콘텐츠 영역                 │
  │  (고정 폭)  │                              │
  └─────────────┴──────────────────────────────┘

lg 미만 (모바일):
  ┌──────────────────────────────────────────┐
  │  상단 서브메뉴 (수평 탭)                 │
  ├──────────────────────────────────────────┤
  │  콘텐츠 영역                             │
  └──────────────────────────────────────────┘
```

### 4.3 메뉴 구성

| pageId | 라벨 |
|---|---|
| `myOrder` | 주문내역 |
| `myClaim` | 취소·반품·교환 |
| `myCoupon` | 쿠폰함 |
| `myCache` | 캐시·적립금 |
| `myContact` | 1:1 문의 |
| `myChatt` | 채팅상담 |

현재 활성 메뉴: accent 색상 강조 + 폰트 굵게.

## 5. 화면 표준

### 5.1 기본 레이아웃

```
full-width (max-width 없음, 좌우 여백만 적용)
  ┌─────────────────────────────────────────────┐
  │  [페이지 콘텐츠]                            │
  │   카드(card), 섹션, 상품 그리드 등          │
  └─────────────────────────────────────────────┘
```

- 콘텐츠 최대 폭은 각 페이지 컴포넌트에서 자체 제어 (`max-w-*` 또는 container 클래스)
- 좌우 패딩: `px-4` (모바일) / `px-6~8` (데스크톱)

### 5.2 반응형 Breakpoints

| 구간 | 최소폭 | 주요 변화 |
|---|---|---|
| 모바일 | 기본 | 1열 레이아웃, 햄버거 메뉴 |
| 태블릿 | `md` (768px) | 2열 그리드, 일부 사이드 패널 표시 |
| 데스크톱 | `lg` (1024px) | 3~4열 그리드, 마이페이지 2컬럼 |
| 와이드 | `xl` (1280px) | 4열 상품 그리드 |

### 5.3 카드 컨테이너

사용자 페이스 카드 스타일: 흰 배경 + 라운드 + 그림자 (`--bg-card`, `--shadow`)

```html
<div class="card">
  <!-- 상품 카드, 주문 카드, 리뷰 카드 등 -->
</div>
```

## 6. 네비게이션 (해시 라우팅)

### 6.1 URL 구조

```
http://127.0.0.1:5501/#page=prodList&pid=123
```

- `page` : 페이지 ID (필수)
- `pid` : 상품/이벤트/블로그 등 항목 ID (선택)

URL 단축: `/index.html` → `/` (history.replaceState, 해시/쿼리 유지)

### 6.2 유효 페이지 ID

| 카테고리 | pageId |
|---|---|
| 메인 | `home` |
| 상품 | `prodList`, `prodView` |
| 구매 | `cart`, `order` |
| 정보 | `contact`, `faq`, `event`, `eventView`, `blog`, `blogView`, `blogEdit` |
| 유틸 | `like`, `location`, `about` |
| 마이페이지 | `myOrder`, `myClaim`, `myCoupon`, `myCache`, `myContact`, `myChatt` |
| 에러 | `error401`, `error404`, `error500` |

### 6.3 navigate 함수

```js
// base/frontApp.js
navigate('prodView', { pid: 123 });
// → #page=prodView&pid=123
```

## 7. 접근 제어

### 7.1 로그인 필요 페이지

`myOrder`, `myClaim`, `myCoupon`, `myCache`, `myContact`, `myChatt`, `order`, `blogEdit`

미로그인 접근 시 → `error401` 페이지로 리다이렉트 또는 로그인 모달 표시

### 7.2 라우터 처리

```js
// base/frontApp.js
if (AUTH_REQUIRED_PAGES.includes(page.value) && !window.frontAuth.state.isLoggedIn) {
  navigate('error401');
  return;
}
```

## 8. 상품 목록 (ProdList)

### 8.1 검색·필터

- 키워드 검색 (상품명/태그)
- 카테고리 필터 (드롭다운 또는 사이드 패널)
- 가격 범위 슬라이더
- 정렬: 최신순 / 인기순 / 낮은가격순 / 높은가격순

### 8.2 목록 표시

- 기본: 그리드 2열 (모바일) / 3~4열 (데스크톱)
- 상품 카드: 이미지 + 브랜드 + 상품명 + 판매가 + 할인율 + 찜 버튼
- 재고 없음: 회색 처리 + "품절" 오버레이

### 8.3 페이지네이션

- 더보기 버튼 방식 또는 번호 페이지네이션
- 기본 page size: 20개

## 9. 상품 상세 (ProdView)

### 9.1 구성 요소

| 영역 | 내용 |
|---|---|
| 이미지 갤러리 | 메인 이미지 + 썸네일 슬라이더 |
| 상품정보 | 브랜드·상품명·옵션·수량·가격·배송비 |
| 구매 버튼 | 장바구니 담기 / 바로구매 |
| 찜 | ♡ 토글 (로그인 필요) |
| 탭 콘텐츠 | 상품설명 / 배송·반품 안내 / 리뷰 / Q&A |

### 9.2 옵션 선택

- 1단 옵션: 단일 select
- 2단 옵션: 1단 select → 2단 select 동적 연동
- 옵션 품절: disabled + "(품절)" 표시

### 9.3 수량 입력

- min=1, max=재고수량
- 직접 입력 또는 +/- 버튼

## 10. 장바구니 (Cart)

### 10.1 표시 정보

- 상품 이미지 + 이름 + 옵션 + 수량(수정 가능) + 단가 + 소계
- 전체선택 / 선택삭제
- 주문금액 합계 카드 (상품합계 + 배송비 + 할인 = 결제예정금액)

### 10.2 쿠폰·캐시 적용

- 쿠폰 적용 모달 (사용 가능 쿠폰 목록)
- 캐시 전액/일부 사용 입력

## 11. 주문 (Order)

### 11.1 주문 프로세스

```
장바구니 → 주문서 작성 → 결제수단 선택 → 결제 완료
```

### 11.2 주문서 구성

1. **배송지 정보** — 주소록 선택 또는 직접 입력 (Kakao 우편번호)
2. **배송 옵션** — 일반·빠른배송·예약배송
3. **주문 상품** — 수량·금액 확인 (수정 불가)
4. **결제수단** — 무통장입금 / 가상계좌 / 토스 / 카카오페이 / 네이버페이 / 핸드폰결제
5. **쿠폰·캐시** — 최종 확인
6. **구매 동의** — 필수 약관 체크박스

### 11.3 결제 완료

- 주문번호 표시
- 주문 내역 확인 → `myOrder` 페이지 이동
