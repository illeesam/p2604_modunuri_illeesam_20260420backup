# pd.06. 세트상품 정책

## 정의
여러 품목을 **하나의 상품 단위**로 묶어 단일 가격에 판매하는 상품 유형.
구성품은 개별 판매하지 않으며, 세트 단위로만 구매·클레임 가능.

> `pd_prod.prod_type_cd = 'SET'`

---

## 묶음상품과 차이

| 구분 | 묶음상품 (GROUP) | 세트상품 (SET) |
|---|---|---|
| 구성품 개별 판매 | 가능 | 불가 |
| 주문 단위 | 구성품 각각 order_item | 세트 자체 1개 order_item |
| 부분 클레임 | 구성품별 가능 | 불가 (세트 전체만) |
| 배송 | 업체별 분리 가능 | 세트 단일 배송 |
| 재고 관리 | 구성품 재고 | 세트 자체 재고 |
| 환불 계산 | price_rate 안분 | 세트 전체 금액 |

---

## 구성 구조

```
pd_prod (세트상품, prod_type_cd=SET)
└─ pd_prod_set_item (구성 목록 — 표시용)
     ├─ item_nm: "머그컵"  qty: 1
     ├─ item_nm: "접시"    qty: 2
     └─ item_nm: "티스푼"  qty: 4
```

> `pd_prod_set_item`은 **표시 목적**. 구성품이 별도 상품 ID를 가질 수도, 안 가질 수도 있음.

---

## 가격·재고 정책

- **단일가**: 세트 자체에 `sale_price` 1개 (안분 없음)
- **재고**: `pd_prod.prod_stock` — 세트 단위 재고
- 구성품이 별도 상품 ID를 가질 경우 구성품 재고도 별도 차감 가능 (선택)

---

## 배송 정책

- 세트 자체의 `dliv_tmplt_id` 적용 → `od_dliv` **1개** 생성
- 세트 내 구성품 출처(업체)가 달라도 배송 분리 없음 — 세트를 하나의 상품으로 취급

---

## 혜택 제한

| 혜택 | 적용 |
|---|---|
| 쿠폰 | 세트 단위 (coupon_use_yn) |
| 적립금 사용 | 세트 단위 |
| 즉시할인 | 세트 단위 |

---

## 주문 후 주문에서 보이는 관점

세트상품 1건 주문 = `od_order_item` **1행** 생성. 구성품 목록은 `od_claim_set_prd`가 아닌
주문 시 `od_order_item_set_detail`(또는 스냅샷 JSON)로 보관한다.

### od_order_item 행 구성
```
od_order_item
  prod_id       : S-PROD-001          ← 세트상품 ID
  prod_nm       : 홈케어 머그세트 (스냅샷)
  prod_type_cd  : SET
  unit_price    : 35,000원            ← 세트 판매가 전체
  qty           : 1
  set_detail    : [                   ← 구성품 스냅샷 (JSON)
    { item_nm: "머그컵",  qty: 1 },
    { item_nm: "접시",    qty: 2 },
    { item_nm: "티스푼",  qty: 4 }
  ]
```

### 주문 화면 표시
- **단일 행**으로 표시 (구성품은 "구성품 보기" 펼치기로 노출)
- 배송 1건 (`od_dliv` 1개)
- 세트 전체 상태 1개로 관리 (구성품별 상태 없음)

---

## 클레임 신청 관점

세트상품은 **세트 전체 단위**로만 클레임 가능. 구성품 1개만 불량이어도 세트 전체를 클레임한다.

### 클레임 가능 단위
| 클레임 유형 | 단위 | 비고 |
|---|---|---|
| 취소 | 세트 전체 | 배송 전만 가능 |
| 반품 | 세트 전체 | 구성품 부분 반품 불가 |
| 교환 | 세트 전체 | 동일 세트로 교환 |

> 구성품 1개 불량 → 세트 전체 반품 처리 → `od_claim_set_prd`에 구성품 명세 기록

### 클레임 접수 흐름
```
세트상품 반품 신청
  → od_claim 생성 (claim_type = RETURN, 세트상품 기준)
  → od_claim_set_prd: 구성품 목록 명세 기록
       clm_no, prd_no, prd_nm, qty, img_url
  → 반품지: 세트상품의 dliv_tmplt.return_addr 기준
  → 반품배송비: 세트상품의 dliv_tmplt.return_cost 기준 (1건)
```

### 교환 시 구성품 변경
```
세트상품 교환 신청 (일부 구성품 불량)
  → od_claim_set_prd에 new_stck_no (교환 구성품 재고번호) 기록
  → 새 세트 출고 또는 불량 구성품만 재발송 (운영 결정)
```

---

## 환불 처리 시 계산 관점

세트상품은 안분이 없으므로 **세트 전체 결제금액 기준** 환불.

### 환불 계산식
```
환불액 = 세트 결제금액
       − 반품배송비 (고객귀책 시)
       − add_shipping_fee (무료배송 조건 파괴 시, 단 세트 전체 반품이면 해당 없음)
```

### 세트 전체 취소 시
```
환불액 = 세트 결제금액 전액
       배송비: 취소 시점 배송 미발생 → 환급 없음 (미부과)
```

## 데이터 시뮬레이션

### 시나리오 전제
```
[세트상품 설정]
prod_id      : S-PROD-001
prod_nm      : 홈케어 머그세트
sale_price   : 35,000원

[구성품 (표시용)]
머그컵 × 1,  접시 × 2,  티스푼 × 4

[배송템플릿]
dliv_cost         : 3,000원
free_dliv_min_amt : 30,000원
return_cost       : 3,000원
exchange_cost     : 6,000원
```

---

### 주문 생성 결과
```
[주문]
ord_id      : ORD-2604-002
prod_amt    : 35,000원
dliv_cost   :      0원   ← 35,000 ≥ 30,000 → 무료배송
pay_amt     : 35,000원

[od_order_item - 1행만 생성]
prod_id      : S-PROD-001
prod_nm      : 홈케어 머그세트
prod_type_cd : SET
unit_price   : 35,000원
qty          : 1
```

---

### 시나리오A: 세트 전체 반품 (고객귀책)
```
반품 대상: 세트 전체
반품 후 잔여: 0원 → 주문 자체 소멸 (무료배송 조건 파괴 개념 미적용)

환불액 계산:
  세트 결제금액   35,000원
− add_shipping_fee      0원  (전체 반품 → 해당 없음)
− return_cost       3,000원  (고객귀책)
= 환불액           32,000원
```

---

### 시나리오B: 세트 전체 반품 (판매자귀책 - 불량)
```
반품 대상: 세트 전체 (머그컵 균열 불량)
귀책: 판매자

환불액 계산:
  세트 결제금액   35,000원
− return_cost           0원  (판매자 귀책 → 면제)
= 환불액           35,000원
```

---

### 시나리오C: 교환 신청 (고객귀책 - 단순변심)
```
교환 대상: 동일 세트 (옵션 없으므로 동일 상품 재발송)

정산:
  추가 결제   6,000원  (exchange_cost 왕복)
  환불        없음
```

---

### 시나리오D: 세트 전체 취소 (배송 전)
```
취소 시점: 결제 완료 직후 (미출고)

환불액 = 35,000원 (결제금액 전액)
배송비 = 0원 (미발생)
```

---

## 제약사항

- 세트 부분 취소·부분 반품·부분 교환 **불가** — 세트 전체 단위로만 클레임
- 세트 구성 변경 시: 기존 주문(스냅샷)에는 영향 없음 (`od_order_item` 스냅샷 유지)
- 구성품 재고 부족 시 세트상품 출고 불가 → `sold_out_yn = 'Y'` 처리
