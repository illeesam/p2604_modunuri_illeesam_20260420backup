# 311. 카테고리 관리 정책

## 목적
상품 분류 체계 구축 및 카테고리 관리 정책 정의

## 범위
- 카테고리 구조 설계
- 카테고리 상태 관리
- 카테고리 깊이 제한

## 카테고리 상태 (USE_YN)
| 상태 | 코드 | 설명 |
|------|------|------|
| 활성 | Y | 사용 중인 카테고리 |
| 비활성 | N | 비활성 카테고리 |

## 카테고리 깊이 (category_depth)
| 깊이 | 명칭 | 역할 |
|------|------|------|
| 1 | 대분류 | 최상위 카테고리 |
| 2 | 중분류 | 대분류 하위 |
| 3 | 소분류 | 중분류 하위 |

## 주요 정책

### 1. 카테고리 구조
- **최대깊이**: 3단계 (대, 중, 소)
- **최소요구**: 1단계 이상
- **대분류**: 최대 20개
- **중분류**: 대분류당 최대 50개
- **소분류**: 중분류당 최대 100개

### 2. 카테고리 생성
- **필수항목**:
  - 카테고리명 (20자 이내)
  - 상위 카테고리ID (깊이 2 이상)
  - 정렬순서
- **기본상태**: Y (활성)
- **이미지**: 선택사항

### 3. 카테고리 운영
- **상품 할당**: 하나의 상품은 소분류에만 할당
- **이동**: 같은 깊이 내에서만 이동 가능
- **삭제**: 하위 카테고리/상품 없을 때만 가능
  - 삭제 전 상품 이동 필수

### 4. 카테고리 활성화
- **활성화**: Y (상품 노출, 검색 가능)
- **비활성화**: N (상품 노출 안됨)
  - 기존 상품은 숨김 처리
  - 신규 상품 등록 불가
  - 비활성 기간 최대 90일

### 5. 정렬순서
- **범위**: 0 ~ 999
- **오름차순 정렬**: 작은 값이 상위 표시
- **동일 순서**: 등록일순

## 주요 필드
| 필드 | 설명 | 규칙 |
|------|------|------|
| category_id | 카테고리ID | YYMMDDhhmmss+rand4 |
| parent_category_id | 상위카테고리ID | NULL이면 대분류 |
| category_nm | 카테고리명 | 20자 이내 필수 |
| category_depth | 깊이 | SMALLINT 1~3 |
| category_status_cd | 상태 | Y/N |
| category_status_cd_before | 변경전상태 | 상태변경 추적 |
| sort_ord | 정렬순서 | INTEGER 기본값 0 |
| img_url | 이미지URL | 선택사항 |
| category_desc | 설명 | TEXT |

## 관련 테이블
- pd_category: 카테고리 정보
- pd_prod: 상품-카테고리 연결

## 제약사항
- 대분류는 삭제 불가 (비활성화만 가능)
- 순환참조 방지 (자신을 상위로 설정 불가)
- 깊이 4 이상 생성 불가
