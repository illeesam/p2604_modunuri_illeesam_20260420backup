# pd.07. 사은상품 정책 (상품 관점)

## 정의
주문 조건(금액·수량·등급) 충족 시 **무상**으로 제공하는 품목.
프로모션 도메인(`pm_gift`)에서 지급 조건을 관리하고, 상품 도메인에서 재고와 상품 연결을 관리한다.

> 지급 조건·이력 정책 → `pm.06.프로모션사은품.md` 참조

---

## 사은상품 유형

| 유형 | 코드 | 설명 | 상품 연결 |
|---|---|---|---|
| 상품 사은품 | PRODUCT | 실제 판매 상품을 사은품으로 지급 | `pm_gift.prod_id` 연결 |
| 샘플 사은품 | SAMPLE | 별도 판매 없는 샘플 품목 | prod_id 없음 |
| 기타 | ETC | 소모품·패키지 등 | prod_id 없음 |

---

## 재고 관리

- **재고 필드**: `pm_gift.gift_stock`
- **차감 시점**: 주문 결제 완료 후 `pm_gift_issue` 생성 시
- **품절 처리**: `gift_stock = 0` → 사은품 지급 불가, 주문 자체는 계속 진행
- 상품 사은품(`PRODUCT`)의 경우 `pd_prod.prod_stock`도 함께 차감 여부는 운영 정책으로 결정

---

## 주문 후 주문에서 보이는 관점

사은상품은 `od_order_item`에 **별도 행**으로 추가되며, `unit_price = 0` 으로 표시된다.
`gift_id`로 발급 건과 연결되어 주문 취소·반품 시 자동 처리 트리거로 활용된다.

### od_order_item 행 구성
```
[일반 주문 상품]
od_order_item
  prod_id     : A-PROD-001
  prod_nm     : 핸드워시 300ml (스냅샷)
  unit_price  : 25,000원
  qty         : 1
  gift_id     : null

[사은상품 행]
od_order_item
  prod_id     : G-PROD-001   (또는 null - SAMPLE/ETC 유형)
  prod_nm     : 미니 �핸드크림 30ml (스냅샷)
  unit_price  :      0원     ← 반드시 0원
  qty         : 1
  gift_id     : GIFT-001     ← pm_gift_issue 연결
  gift_type   : PRODUCT
```

### 주문 화면 표시
- 사은상품 행은 "🎁 사은품" 배지 표시
- 결제금액 합산에서 제외 (`unit_price = 0`)
- 주문 상품 목록 하단 또는 별도 섹션으로 구분 표시 권장
- 품절로 미지급된 경우: "사은품 품절" 안내 문구 표시

## 클레임 신청 관점

사은상품은 단독으로 취소·반품 클레임을 신청할 수 없다.
**주문 상품의 클레임 연동**으로만 처리된다.

### 클레임 유형별 처리
| 클레임 유형 | 사은상품 처리 | 비고 |
|---|---|---|
| 주문 전체 취소 | 자동 취소 | `pm_gift_issue.status = CANCELLED` + 재고 복원 |
| 일부 취소 (조건 유지) | 사은품 유지 | 잔여 주문금액 ≥ 지급 조건 |
| 일부 취소 (조건 파괴) | 회수 또는 시가 청구 | 잔여 금액 < 지급 조건 |
| 반품 (조건 유지) | 사은품 유지 | |
| 반품 (조건 파괴) | 회수 또는 시가 청구 | 반품 상품 포함 여부 확인 |
| 교환 | 사은품 유지 | 교환은 상품 변경이므로 지급 조건 재판단 |

### 조건 파괴 판단
```
지급 조건: 주문금액 30,000원 이상 → 사은품 지급
클레임 후 잔여 주문금액 < 30,000원 → 조건 파괴
  → 사은상품 회수 요청 (반품 택배에 동봉 요청)
     OR 시가(pm_gift.gift_price) 청구 → 환불 금액에서 차감
```

### 사은상품 회수 vs 시가 청구 선택 기준
| 상황 | 처리 방법 |
|---|---|
| 미개봉 상태 | 반품 택배 동봉 회수 |
| 개봉·사용 | 시가(`gift_price`) 청구 |
| 샘플·소모품(ETC) | 시가 청구 (회수 불필요) |
| 운영 정책 우선 | 업체별 정책 따름 |

---

## 환불 처리 시 계산 관점

사은상품 자체는 `unit_price = 0`이므로 환불 계산에 직접 포함되지 않는다.
단, **조건 파괴 시 시가 청구액**이 환불 금액에서 차감된다.

### 환불 계산식
```
[조건 파괴 없음]
환불액 = 반품 상품 결제금액 − 반품배송비 (고객귀책 시)

[조건 파괴 + 시가 청구]
환불액 = 반품 상품 결제금액
       − 반품배송비 (고객귀책 시)
       − pm_gift.gift_price × 미회수 수량   (사은품 시가)

[사은품 동봉 회수 확인 시]
환불액 = 반품 상품 결제금액 − 반품배송비 (회수 확인 → 시가 청구 없음)
```

## 데이터 시뮬레이션

### 시나리오 전제
```
[주문]
ord_id       : ORD-2604-003
상품A        : 핸드워시 300ml   25,000원
상품B        : 바디로션 200ml   15,000원
주문금액 합계: 40,000원
dliv_cost    :      0원   (30,000원 이상 무료)
pay_amt      : 40,000원

[사은품 지급 조건 - pm_gift]
gift_id      : GIFT-001
gift_nm      : 미니 핸드크림 30ml
gift_price   : 5,000원    ← 시가 (환불 차감용)
gift_cond    : 주문금액 30,000원 이상
gift_stock   : 50개

[배송템플릿]
return_cost       : 3,000원
free_dliv_min_amt : 30,000원
```

---

### 주문 생성 결과
```
[od_order_item]
행1  핸드워시 300ml   unit_price=25,000  gift_id=null
행2  바디로션 200ml   unit_price=15,000  gift_id=null
행3  미니 핸드크림    unit_price=0       gift_id=GIFT-001  ← 사은상품 행

[pm_gift_issue]
gift_issue_id : GI-001
order_id      : ORD-2604-003
gift_id       : GIFT-001
status        : ISSUED
gift_stock    : 49개 (1개 차감)
```

---

### 시나리오A: 상품A 취소 → 조건 파괴 + 시가 청구
```
취소 상품: 핸드워시 25,000원
취소 후 잔여: 바디로션 15,000원  (< 30,000원 → 조건 파괴!)

환불액 계산:
  취소 상품금액   25,000원
− gift_price       5,000원   (사은품 시가 청구)
= 환불액           20,000원

pm_gift_issue.status → CANCELLED (재고 복원: 50개)
```

---

### 시나리오B: 전체 취소
```
취소 상품: 전체 (주문 전체 취소)

사은품 자동 취소 (pm_gift_issue.status → CANCELLED)
재고 복원: gift_stock = 50개

환불액 = 40,000원 (전액)
사은품 시가 청구 없음 (전체 취소이므로 회수 또는 미사용 기준 처리)
```

---

### 시나리오C: 상품B 반품 → 조건 파괴 + 시가 청구 (고객귀책)
```
반품 상품: 바디로션 15,000원
반품 후 잔여: 핸드워시 25,000원  (< 30,000원 → 조건 파괴!)
귀책: 고객

환불액 계산:
  반품 상품금액   15,000원
− return_cost      3,000원  (고객귀책 반품배송비)
− gift_price       5,000원  (사은품 시가 청구)
= 환불액            7,000원
```

---

### 시나리오D: 교환 신청 → 사은품 유지
```
교환 상품: 바디로션 (색상 변경)
교환 후 잔여 주문금액: 동일 (교환은 상품 대체이므로 금액 변동 없음)

처리:
  사은품 유지 (pm_gift_issue.status = ISSUED 유지)
  추가 정산 없음 (교환 배송비는 별도)
```
